## 1. 产品概述

- **产品名称**：紫微灵犀
- **一句话定位**：基于紫微斗数排盘引擎与大语言模型（Gemini）的 AI 国学文化解读平台
- **目标平台**：微信小程序（主力）+ H5（引流），基于 uni-app 一套代码多端发布
- **目标用户**：对紫微斗数感兴趣、希望获得 AI 命理解读的 C 端用户（18-45 岁，移动端优先）
- **核心价值**：将传统紫微斗数排盘与 AI 深度解读结合，用户输入生辰信息后自动排盘，通过积分解锁不同维度的提示词模板获取个性化命理分析报告
- **合规定位**：传统文化知识服务 + AI 内容生成，所有解读内容标注"仅供娱乐参考"

### 1.1 产品背景

紫微斗数是中国传统命理学的核心体系之一，排盘流程专业、解读门槛高。当前市场上的同类产品（神机阁、文墨天机、紫微派等）多数仅提供排盘工具或固定模板的文字解读，AI 深度解读能力薄弱、个性化程度低。本产品利用已有的 FastAPI 排盘引擎（基于 iztro-py 计算库）和 Gemini 大模型，实现「排盘 + AI 深度解读」的闭环体验。

### 1.2 商业模式（三阶段演进）

| 阶段 | 策略 | 收入来源 | 所需资质 |
|------|------|---------|---------|
| **阶段一（MVP）** | 免费 + 积分体验 | 无直接收入（验证产品） | 个人主体微信小程序 |
| **阶段二（有用户后）** | 积分 + 广告变现 | 激励视频广告（看广告得积分） | 小程序日活 > 100 即可开通广告 |
| **阶段三（规模化）** | 订阅付费 + 广告 | 月度/年度订阅 + 广告收入 | 注册个体工商户，升级企业主体 |

### 1.3 现有技术资产

| 资产 | 说明 |
|------|------|
| 排盘 API | FastAPI 服务，已实现 `GET /paipan`（农历）、`GET /paipan/solar`（阳历）、`GET /paipan/json`（JSON 格式） |
| 计算引擎 | iztro-py v0.3.4，支持十二宫排列、主星/辅星/小星/神煞、大限/小限/流年计算 |
| 输出格式 | 树形文本格式 + 结构化 JSON（含 palaces、stars、decadal、ages 等完整数据） |

| **部署** | 待定 | 后端部署方案开发前确定 |

## 3. 功能需求

### 3.1 用户注册与登录

- **描述**：用户通过手机号 + 密码注册/登录系统，首次注册自动赠送初始积分，登录后必须完善个人生辰信息才能使用排盘和 AI 解读功能
- **用户故事**：作为新用户，我希望快速注册并绑定我的生辰信息，以便系统为我生成紫微命盘
- **验收标准**：
  - AC-1: Given 用户首次访问，When 输入手机号和密码点击注册，Then 注册成功并自动登录，**系统赠送 100 积分**，跳转至生辰信息完善页
  - AC-2: Given 用户在生辰信息完善页，When 填写完出生日期（支持阳历/农历切换）、时辰（13 个选项：早子时至晚子时）、性别、出生地、是否闰月，Then 信息保存成功，跳转至首页
  - AC-3: Given 已注册用户再次访问，When 输入手机号和密码登录，Then 直接登录跳转首页
  - AC-4: Given 用户未完善生辰信息，When 尝试访问排盘或 AI 解读功能，Then 强制跳转至生辰信息完善页
  - AC-5: Given 用户已完善信息，When 进入「个人中心」，Then 可查看和修改生辰信息（修改后需重新排盘）
- **优先级**：P0
- **边界条件与异常场景**：
  - 手机号格式校验：仅接受中国大陆 11 位手机号，手机号唯一
  - 密码最少 8 位
  - 注册时手机号已存在则提示"该手机号已注册，请直接登录"
  - 阳历日期范围限制：1900-01-01 至当天
  - 时辰选择为必填项，不可留空
  - 出生地为选填项，精确到省/市级别
  - 农历闰月选项仅在选择农历输入时显示
  - 修改生辰信息后，历史排盘数据保留，新排盘覆盖当前命盘
  - 注册赠送积分仅限首次注册，同一手机号不可重复领取

### 3.2 紫微斗数排盘

- **描述**：系统根据用户绑定的生辰信息，调用现有排盘 API（`/paipan/json` 或 `/paipan/solar`）生成紫微命盘数据，并以可视化方式展示
- **用户故事**：作为已完善信息的用户，我希望系统自动为我排盘并清晰展示命盘结果，以便我了解自己的十二宫配置
- **验收标准**：
  - AC-1: Given 用户完善生辰信息后首次进入首页，When 系统检测到该用户尚无排盘数据，Then 自动调用排盘 API 并将结果存储至用户档案
  - AC-2: Given 排盘完成，When 用户进入「我的命盘」页面，Then 展示十二宫方位图，每宫显示：宫名、天干地支、主星（含亮度和四化标注）、辅星、大限年龄范围
  - AC-3: Given 用户点击某个宫位，When 宫位详情展开，Then 展示该宫完整信息：主星、辅星、小星、神煞（岁前星/将前星/十二长生/太岁煞禄）、小限、流年
  - AC-4: Given 用户修改了生辰信息，When 保存修改，Then 系统自动重新调用排盘 API 并更新存储数据
  - AC-5: Given 排盘 API 调用失败（网络超时/服务异常），When 发生错误，Then 展示友好错误提示并提供「重试」按钮
- **优先级**：P0
- **边界条件与异常场景**：
  - 排盘 API 超时时间设定为 10 秒，超时后提示用户重试
  - 排盘结果缓存于数据库，同一用户同一生辰信息不重复调用 API
  - 命盘数据包含基本信息（农历时间、四柱、五行局数、身主、命主、斗君、身宫）和十二宫详细数据
  - API 参数映射：阳历输入调用 `/paipan/solar`（date=YYYY-M-D 格式），农历输入调用 `/paipan/json`（year/month/day 分离参数 + is_leap_month）
  - time_index 范围 0-12，前端时辰选择器需正确映射

### 3.3 积分系统

- **描述**：积分是平台的核心虚拟货币，用户通过注册赠送、分享奖励、观看广告等方式获取积分，消耗积分解锁模板和使用 AI 解读
- **用户故事**：作为用户，我希望通过多种方式获取积分，以便解锁更多模板获取 AI 解读
- **验收标准**：
  - AC-1: Given 新用户注册成功，When 系统处理注册逻辑，Then 自动向用户账户充入 100 积分，并记录积分流水（类型：注册赠送）
  - AC-2: Given 用户在任意页面，When 查看积分余额，Then 顶部导航栏/个人中心始终显示当前积分余额
  - AC-3: Given 用户积分不足以解锁某模板，When 点击解锁按钮，Then 弹出提示"积分不足"，并展示获取积分的途径入口（分享得积分、看广告得积分）
  - AC-4: Given 用户成功分享小程序到微信聊天/群，When 分享回调确认，Then 奖励 10 积分，每日分享奖励上限 30 积分（3 次）
  - AC-5: Given 用户点击「看广告得积分」，When 完整观看一条激励视频广告，Then 奖励 20 积分，每日广告奖励上限 100 积分（5 次）
  - AC-6: Given 用户在个人中心点击「积分明细」，When 进入积分流水页，Then 按时间倒序展示所有积分变动记录（时间、类型、变动数量、余额）
- **优先级**：P0
- **积分经济设计**：

| 获取途径 | 单次积分 | 每日上限 | 阶段 |
|---------|---------|---------|------|
| 注册赠送 | 100 | 仅一次 | 阶段一 |
| 分享小程序 | 10 | 30（3次） | 阶段一 |
| 观看激励视频广告 | 20 | 100（5次） | 阶段二（日活>100后开通） |
| 连续签到 | 5-20（递增） | 一次/天 | 阶段一 |
| 邀请新用户注册 | 50 | 无上限 | 阶段一 |

| 消耗场景 | 消耗积分 | 说明 |
|---------|---------|------|
| 解锁免费模板 | 0 | 免费模板不消耗积分 |
| 解锁普通模板 | 30-50 | 单次解锁，永久可用 |
| 解锁高级模板 | 80-150 | 单次解锁，永久可用 |
| AI 解读（每次） | 10 | 每次使用已解锁模板进行 AI 解读时消耗 |

- **边界条件与异常场景**：
  - 积分不可提现、不可转让、不可交易
  - 积分无有效期（MVP 阶段不设过期机制）
  - 积分余额不可为负数，消耗前必须校验余额
  - 所有积分变动必须记录流水，支持审计
  - 分享奖励需验证真实分享行为（微信分享回调），防止刷积分
  - 广告奖励需验证广告完整播放（广告 SDK 回调确认）
  - 积分数值设计可后台动态配置，便于运营调整

### 3.4 模板商城

- **描述**：展示后台配置的各类命理解读模板，用户浏览、预览，使用积分解锁模板后方可用于 AI 解读
- **用户故事**：作为用户，我希望浏览不同维度的命理分析模板并按需解锁，以便有针对性地获取 AI 解读
- **验收标准**：
  - AC-1: Given 用户进入模板商城页，When 页面加载完成，Then 以卡片列表形式展示所有上架模板，每张卡片包含：模板名称、简介（50 字以内）、适用标签（如「事业」「感情」「流年」）、**所需积分**、已解锁人数
  - AC-2: Given 用户点击某个模板卡片，When 进入模板详情页，Then 展示：模板名称、详细说明（解读维度和内容概要）、示例解读截图/预览（脱敏处理）、**所需积分**、「积分解锁」按钮
  - AC-3: Given 用户点击「积分解锁」且积分充足，When 确认解锁，Then 扣除对应积分，模板状态变为「已解锁」，记录积分流水
  - AC-4: Given 用户点击「积分解锁」但积分不足，When 弹出提示，Then 展示"积分不足，还差 X 积分"，并提供获取积分途径入口
  - AC-5: Given 用户已解锁某模板，When 在商城浏览该模板，Then 卡片和详情页均显示「已解锁」标识，不重复扣费
  - AC-6: Given 模板列表超过 10 个，When 用户滚动页面，Then 支持下拉加载更多（分页加载，每页 10 个）
- **优先级**：P0
- **边界条件与异常场景**：
  - 模板积分为 0 时，视为免费模板，用户无需消耗积分直接解锁
  - 模板下架后，已解锁用户仍可使用；未解锁用户不可见
  - 单个模板解锁后永久有效（MVP 阶段不做有效期限制）
  - 积分扣除与模板解锁必须在同一事务中完成，避免扣积分但未解锁的情况

### 3.5 后台管理 -- 提示词模板 CRUD

- **描述**：管理员通过后台管理界面对提示词模板进行创建、编辑、上下架、积分定价管理
- **用户故事**：作为管理员，我希望灵活管理提示词模板库（增删改查、定价、上下架），以便控制商城展示内容和运营策略
- **验收标准**：
  - AC-1: Given 管理员登录后台，When 进入「模板管理」页面，Then 展示模板列表表格：ID、名称、标签、**所需积分**、状态（上架/下架）、解锁人数、创建时间、操作栏
  - AC-2: Given 管理员点击「新建模板」，When 填写模板信息（名称、简介、详细说明、提示词正文、标签、**所需积分**、示例解读预览），Then 模板创建成功，默认状态为「下架」
  - AC-3: Given 管理员编辑已上架模板的提示词正文，When 保存修改，Then 修改立即生效，已解锁用户下次使用该模板时将获得更新后的提示词解读
  - AC-4: Given 管理员将模板状态切换为「上架」，When 操作成功，Then 该模板在前端商城可见
  - AC-5: Given 管理员将模板状态切换为「下架」，When 操作成功，Then 该模板在前端商城不可见，已解锁用户的「已解锁」列表中仍保留
- **优先级**：P0
- **边界条件与异常场景**：
  - 提示词正文字段支持使用占位符变量：`{{排盘数据}}`、`{{用户性别}}`、`{{用户出生信息}}`，系统在调用 AI 时自动替换
  - 模板名称不可重复
  - 积分字段为非负整数，0 表示免费
  - 删除操作为软删除，已解锁记录保留
  - 后台管理须通过独立的管理员账号密码登录，与前端用户系统隔离

### 3.6 AI 智能解读

- **描述**：用户选择已解锁的提示词模板后，消耗积分，系统将模板提示词与用户排盘数据拼接，调用 Gemini API 生成个性化命理解读，支持流式输出
- **用户故事**：作为用户，我希望选择一个模板后立即获得基于我命盘数据的 AI 深度解读，以便了解自己在该维度的命理分析
- **验收标准**：
  - AC-1: Given 用户进入「AI 解读」页面，When 展示已解锁模板列表，Then 每个模板以卡片形式展示名称、标签和简介，卡片可点击选择
  - AC-2: Given 用户选择某个模板并点击「开始解读」，When 系统校验积分余额 >= 10（单次解读消耗），Then 扣除 10 积分，将模板提示词中的占位符替换为用户实际排盘数据（JSON 格式），拼接成完整 prompt 发送给 Gemini API
  - AC-3: Given Gemini API 返回流式响应，When 数据逐步到达，Then 前端以打字机效果逐字/逐段展示 AI 解读内容，页面自动向下滚动
  - AC-4: Given AI 解读完成，When 全部内容输出完毕，Then 展示「保存报告」和「分享」按钮，用户可将报告保存至「我的报告」
  - AC-5: Given 用户查看「我的报告」列表，When 点击某份历史报告，Then 完整展示该次 AI 解读内容（含使用的模板名称、生成时间）
  - AC-6: Given 同一用户使用同一模板重复解读，When 生成新报告，Then 新报告追加至历史记录，旧报告保留不覆盖
  - AC-7: Given 用户积分不足 10，When 点击「开始解读」，Then 提示"积分不足"并引导获取积分
- **优先级**：P0
- **边界条件与异常场景**：
  - Gemini API 调用超时上限 60 秒，超时后提示「解读生成超时，请重试」，**已扣除的积分原路退回**
  - Gemini API 返回内容为空或格式异常时，提示「解读生成失败，请稍后重试」并记录日志，**已扣除的积分原路退回**
  - 流式输出过程中用户关闭页面，已接收内容不保存（仅完整生成的报告才可保存），**积分不退回**（已消耗 AI 资源）
  - 流式输出过程中网络中断，展示「网络异常」提示和「重试」按钮
  - 单次 prompt 长度限制：提示词模板上限 2000 字符，排盘数据上限 5000 字符（JSON 格式）
  - AI 解读内容需在开头附加免责声明："以下内容由 AI 基于紫微斗数理论生成，仅供参考娱乐，不构成任何决策建议。"
  - **【v2.1 复核补充 - AIGC 标识合规】** 依据《人工智能生成合成内容标识办法》（2025-09 施行），AI 解读内容必须同时附加**显式标识**：
    - 报告页底部固定展示标识条："本报告由 AI（gemini-2.5-pro）生成，仅供文化娱乐参考"
    - 分享卡片必须包含 AIGC 标识水印（不可被裁剪掉）
    - 报告 metadata 中保留隐式标识（model_name + 生成时间），便于审计
  - **【v2.1 复核补充 - 首次解读体验闭环】** 用户完成生辰信息绑定后的**首次 AI 解读不消耗积分**（仅消耗一次，需有"首次免费"标记字段）：
    - 触发条件：UserProfile 创建后 7 天内的首次 `/api/v1/reading/start` 调用
    - 选择模板范围：仅限免费模板（points_cost = 0），不可用首次免费解锁付费模板
    - 数据模型补充：User 表需增加 `free_reading_used` BOOL 字段，默认 false，首次使用后置 true
    - 设计动机：调研显示用户反感"刚注册就遇到积分不足"的体验，确保首次完整闭环（参见 research-market.md 4.4）

### 3.7 个人中心

- **描述**：用户管理个人账户信息、积分、已解锁模板、历史报告的综合入口
- **用户故事**：作为用户，我希望在一个集中的页面管理我的账户、积分、已解锁内容和历史解读记录
- **验收标准**：
  - AC-1: Given 用户进入个人中心，When 页面加载完成，Then 展示以下分区：基本信息（头像、昵称、手机号）、**积分余额（突出显示）**、生辰信息摘要、快捷入口（已解锁模板数、历史报告数）
  - AC-2: Given 用户点击积分余额区域，When 进入积分中心页，Then 展示：当前余额、获取积分途径（分享、广告、签到）、积分明细流水
  - AC-3: Given 用户点击「我的模板」，When 进入已解锁模板列表，Then 展示所有已解锁的模板，点击可直接跳转至 AI 解读
  - AC-4: Given 用户点击「我的报告」，When 进入历史报告列表，Then 按时间倒序展示所有 AI 解读报告，每条显示模板名称、生成时间、摘要前 50 字
- **优先级**：P0（从 P1 提升，积分中心是核心功能）
- **边界条件与异常场景**：
  - 无已解锁模板时展示空状态引导："还没有解锁模板，去商城看看"
  - 无历史报告时展示空状态引导："还没有 AI 解读记录，选个模板开始吧"
  - 头像和昵称修改功能在 MVP 阶段不实现，展示默认头像和手机号后四位作为昵称

### 3.8 签到系统

- **描述**：用户每日签到获取积分，连续签到可获得递增奖励，提升用户留存
- **用户故事**：作为用户，我希望每天签到就能获得积分，鼓励我持续使用
- **验收标准**：
  - AC-1: Given 用户进入首页，When 今日未签到，Then 展示签到入口（悬浮按钮或横幅），点击后完成签到
  - AC-2: Given 用户点击签到，When 签到成功，Then 展示签到动画和获得积分数量，积分即时到账
  - AC-3: Given 用户连续签到 N 天，When 签到成功，Then 积分按连续天数递增（第1天5分，第2天8分，第3天10分，第4天12分，第5天15分，第6天18分，第7天20分，之后循环）
  - AC-4: Given 用户中断签到（某天未签到），When 次日签到，Then 连续天数重置为 1
- **优先级**：P0
- **边界条件与异常场景**：
  - 签到以自然日（北京时间 0:00 重置）为单位
  - 每日仅可签到一次，重复签到提示"今日已签到"
  - 签到记录和积分变动在同一事务中完成

### 3.9 分享裂变

- **描述**：用户分享小程序/报告至微信，获取积分奖励，同时实现用户增长裂变
- **用户故事**：作为用户，我希望分享有趣的解读给朋友，同时获得积分奖励
- **验收标准**：
  - AC-1: Given 用户在报告详情页点击「分享」，When 触发微信分享，Then 生成分享卡片（报告摘要前 50 字 + 模板名 + 产品 Logo），分享成功后奖励 10 积分
  - AC-2: Given 用户通过首页/个人中心点击「邀请好友」，When 生成专属邀请分享卡片并发送，Then 被邀请用户注册成功后，邀请人获得 50 积分
  - AC-3: Given 被邀请用户通过分享链接打开小程序，When 注册成功，Then 被邀请用户也获得注册赠送积分（100 积分）
- **优先级**：P1
- **边界条件与异常场景**：
  - 分享内容脱敏处理：隐藏用户手机号，生辰信息仅显示年份和性别
  - 分享积分奖励每日上限 30 积分（3 次），防止刷分
  - 邀请奖励需确认被邀请用户为新用户（手机号从未注册过）
  - H5 端分享生成链接（复制到剪贴板），小程序端使用原生分享能力

### 5.2 导航逻辑

- 前端采用底部 Tab 导航（4 个 Tab）：首页、模板商城、AI 解读、个人中心
- 登录/注册为独立全屏页面，登录状态下不可访问
- 生辰信息完善页为强制中间页，未完善时拦截所有核心功能入口
- 管理后台为独立 H5 页面（独立 URL 路径 `/admin`），不在小程序中暴露
- 积分不足时的弹层提供快捷跳转至积分获取途径

### 5.3 核心用户动线

**新用户首次使用动线**：
1. 打开小程序 -> 引导页 -> 登录注册页（输入手机号 + 密码）
2. 注册成功 -> **获得 100 积分** -> 强制跳转生辰信息完善页（填写出生信息）
3. 信息保存 -> 系统自动排盘 -> 跳转首页
4. 首页签到获得积分 -> 浏览推荐模板 -> 点击模板 -> 进入模板详情 -> **积分解锁**
5. 解锁成功 -> 跳转 AI 解读页 -> 选择模板 -> **消耗 10 积分开始解读** -> 流式查看结果
6. 解读完成 -> 保存报告 / **分享得积分**

**老用户复访动线**：
1. 打开小程序 -> 自动登录 -> 首页 -> **签到得积分**
2. 点击「AI 解读」Tab -> 选择已解锁模板 -> 消耗积分开始解读

**积分不足时的动线**：
1. 尝试解锁模板/开始解读 -> 提示积分不足
2. 跳转积分中心 -> 选择获取方式（签到/分享/看广告）
3. 获取足够积分 -> 返回继续操作

## 9. 设计指引

### 9.1 风格方向

- **整体调性**：东方美学 + 现代极简，传达「古典智慧，科技解读」的品牌感
- **色彩基调**：深紫色（#2D1B69）为主色调，金色（#D4A84B）为点缀色，背景色使用暖白（#FFF8F0）或深色模式（#1A1025）
- **字体选择**：标题使用带有书法韵味的中文字体（如思源宋体），正文使用现代无衬线体（如思源黑体/苹方）
- **图形元素**：适当使用星曜图标、天干地支符号、中式线条纹样作为装饰元素
- **命盘可视化**：传统正方形十二宫格排布（4x4 去四角，或标准 3x4 等排列），每宫位内分层显示信息
- **积分元素**：积分图标使用金色圆形（类似古钱币/星币），与东方美学主题统一

- **限制**字体家族数量 ≤ 3（思源宋体 + 思源黑体 + 数字字体，不再引入第 4 个）

### v2.1 复核决策详情

#### Q3 - 初始预设模板清单（共 7 个）

**免费体验模板（2 个，points_cost = 0）**：

| 模板名称 | 标签 | 解读维度 | 预期字数 |
|---------|------|---------|---------|
| 命宫初探 | ["命宫", "性格"] | 命宫主星组合、基本性格特征、人生总基调 | 600-800 字 |
| 今日运势 | ["流日", "运势"] | 基于流日盘的当日宜忌、能量提示 | 400-600 字 |

**付费解锁模板（5 个）**：

| 模板名称 | 标签 | 解读维度 | points_cost |
|---------|------|---------|-------------|
| 事业财禄全景 | ["官禄", "财帛", "事业"] | 官禄宫 + 财帛宫深度联动分析、职业方向、财富格局 | 30 |
| 感情桃花密码 | ["夫妻", "感情"] | 夫妻宫主辅星组合、桃花格局、配偶画像、感情节奏 | 50 |
| 流年运程详批 | ["大限", "流年"] | 当前大限 + 流年盘解读、本命年关键事件预警 | 50 |
| 健康疾厄预警 | ["疾厄", "健康"] | 疾厄宫星曜分析、易发病灶提示、养生建议 | 30 |
| 父母兄弟人脉 | ["父母", "兄弟", "交友"] | 三宫综合分析、家族关系、社交人脉格局 | 30 |

**合计**：免费 2 个 + 付费 5 个 = 7 个，覆盖紫微斗数十二宫的核心 4-5 个高关注宫位。

**理由**：
- 调研显示用户最关心的 3 个维度是事业 / 感情 / 流年，因此付费模板这 3 个定 50 分（高价值）
- 健康和家庭关系作为辅助维度定 30 分（性价比）
- 7 个模板既不过少（满足多次解读复购）也不过多（避免选择困难）

#### Q4 - 积分经济平衡

**保持现有积分配置**（参见 3.3 积分经济设计），并补充：

- **注册赠 100 积分** + **首次解读不消耗积分**（即新用户进来可至少完成 1 次完整闭环：免费模板 + 1 次免费解读 + 还剩 100 积分可解锁付费模板）
- **首次免费解读限免费模板**（避免被薅羊毛解锁高价模板）
- **新手任务包**（建议阶段一上线时同步加入，但**不写入本 PRD MVP 范围**，作为运营活动）：完善生辰信息 +10、首次签到 +10、首次分享 +20，引导熟悉积分获取路径

**度量指标**（上线后跟踪，决定是否调整）：
- 新用户首次解读完成率 > 80%（衡量首次体验闭环）
- 7 日留存 > 25%（衡量积分激励效果）
- 付费模板解锁转化率 > 5%（衡量积分定价合理性）

#### Q7 - AI 解读法律风险三层防护

| 层级 | 措施 | 实施位置 |
|------|------|---------|
| **第一层 - 免责声明** | 每次解读结果开头追加 "以下内容由 AI 基于紫微斗数理论生成，仅供参考娱乐，不构成任何决策建议。" | 后端 prompt 拼接时强制前缀，前端不可关闭 |
| **第二层 - AIGC 标识** | 报告页底部固定标识条 + 分享卡片不可裁剪水印（依据网信办 2025-09《人工智能生成合成内容标识办法》） | 前端组件 / 分享生成器 |
| **第三层 - 内容兜底** | Gemini API 调用前的 prompt 模板加入"避免医疗诊断、法律建议、具体投资建议"硬性约束；返回内容做敏感词过滤 | 后端 reading 服务 |

**版权归属**：
- AI 生成内容**不享有著作权**（依据 2025 年司法实践，纯 AI 生成内容不受著作权保护）
- 用户输入的生辰信息属于个人隐私数据，存储和处理须符合《个人信息保护法》
- 提示词模板属于平台知识产权，不向用户公开（已在 4.2 安全要求中规定）

**算法备案**：
- MVP 阶段（用户 < 1 万）暂不备案，但需在产品页明确标注"本服务使用 Gemini API 生成内容"
- 规模化前（用户 > 5 万 或 月活 > 1 万）必须完成生成式 AI 服务备案（参考 748 款已备案产品基线）

- 不得在小程序名称中出现"算命"、"风水"、"占卜"等敏感词（产品名"紫微灵犀"已规避）
- 准备运营资质：个人身份证 + 手持照（个人主体足够，PRD Q11 已定）

#### Q14 - H5 ICP 备案策略

**MVP 阶段策略**：H5 部署在**已备案域名的子路径**（如 `xxx.com/ziwei/`），无需单独备案。

**实施方案**：
- 选项 A：用户已有备案域名（如个人博客 / 工作室站点），开辟子路径承载 H5（推荐）
- 选项 B：暂不上线 H5，纯小程序运营，待阶段三注册主体后申请独立备案域名
- 选项 C：使用海外服务器 + 海外域名（如 Vercel + custom domain），仅服务海外用户（不推荐，违背"微信生态主战场"定位）

**阶段三规划**：注册个体工商户后申请独立 ICP 备案，迁移 H5 至独立域名（如 ziweilingxi.com）。

**风险提示**：未备案的 H5 不可在国内主流应用市场推广（如不能投信息流广告），仅适合微信内分享传播。
