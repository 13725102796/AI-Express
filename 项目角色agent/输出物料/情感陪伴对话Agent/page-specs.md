## 3. 页面详细规格

---

### [P01] 对话主页

**路由**：/chat
**关联 PRD 功能**：3.2 核心对话引擎, 3.3 语音倾诉, 3.4 渐进式记忆, 3.8 呼吸锚点, 3.9 安全守护
**页面定位**：留白的灵魂页面，用户 80%+ 时间在此。情绪容器，一切设计服务于"安全感 + 被理解感"。

#### 页面结构

- **区域 A - 顶部栏**：固定顶部，高度 56px。左侧：留白 logo（小字）+ AI 在线状态指示（绿色呼吸圆点）；右侧：呼吸锚点入口按钮（「冷静一下」小字链接）
- **区域 B - 对话消息流**：占据中间全部空间，垂直滚动。包含用户消息气泡、AI 消息气泡、时间戳分隔符、情绪化石生成提示、安全守护资源卡片
- **区域 C - 输入区域**：固定底部，safe-area 适配。多行文字输入框 + 语音输入按钮 + 发送按钮。输入框上方可出现"正在输入..."指示
- **区域 D - 呼吸锚点（覆盖层）**：全屏覆盖层，仅在触发呼吸模式时显示。中央呼吸圆 + 底部文字引导 + 右上角关闭按钮
- **区域 E - 化石生成浮层**：对话结束后从底部上浮的情绪化石预览卡片 + "写一封回声信"入口

#### 页面状态

| 状态 | 触发条件 | 页面表现 |
|------|---------|---------|
| 首次空状态 | 新用户从引导页跳转来，无历史对话 | 对话区只有 AI 的开场白："[称呼]，以后我就在这里。想聊什么都可以。" 输入框 placeholder "想说点什么..." |
| 正常对话状态 | 有历史对话 | 消息流展示历史消息 + 新消息，底部输入区就绪 |
| AI 回复中 | 用户发送消息后 | AI 状态指示变为"在想..."，显示三点呼吸动画，随后流式输出文字 |
| 语音录入中 | 用户长按麦克风按钮 | 输入区变为录音模式：显示录音时长、波形动画、"松开结束录音" 提示 |
| 呼吸锚点模式 | 用户点击"我需要冷静一下"或 AI 建议并用户同意 | 全屏覆盖层，深色半透明背景，中央呼吸圆，底部文字引导。右上角淡色关闭按钮 |
| 安全守护触发 | AI 检测到用户可能有自伤/自杀意向 | 对话流中自然嵌入安全守护资源卡片（心理热线信息），不弹窗打断。AI 回复切换为共情+资源呈现模式 |
| 化石生成 | 8 轮以上的深度对话结束后 | 底部浮出情绪化石预览卡 + "写一封回声信"入口，5 秒后可手动关闭 |
| 错误状态 | 网络异常 | AI 回复区显示"网络有点不稳定，等一下再试"，带重试按钮 |
| AI 超时 | AI 回复超过 10 秒 | 显示"在想怎么好好回复你..."占位文字 |
| 回访问候 | 老用户打开留白，距上次对话有间隔 | AI 基于记忆系统主动问候，如"面试怎么样？"或"最近工作是不是又很忙？"，消息自然出现在对话流顶部 |
| 语音时长限制 | 录音达到 120 秒 | 录音自动停止，提示"说了好多呢，先发这些吧"，转写结果填入输入框 |

#### 交互逻辑

1. **I-01-01 发送文字消息**：用户在输入框输入文字 → 点击发送按钮或按 Enter → 消息以用户气泡样式追加到消息流底部（淡入上浮动效）→ 输入框清空 → 显示"留白在想..."指示 → 1-3 秒后 AI 回复开始流式显示 → 流式完成后在 AI 气泡下方淡入情绪感知标签
2. **I-01-02 语音输入**：用户长按麦克风按钮 → 输入区切换为录音模式（波形动画 + 时长计时）→ 松开结束录音 → 显示"处理中..."→ 2 秒内转写结果填入输入框 → 用户确认后点击发送
3. **I-01-03 进入呼吸锚点**：用户点击"我需要冷静一下"→ AI 回复确认"要不要先停一下？我可以陪你做几次深呼吸。"+ 同意/取消按钮 → 用户点击同意 → 对话界面渐变（500ms）为全屏呼吸引导界面 → 呼吸圆开始循环动画
4. **I-01-04 退出呼吸锚点**：呼吸引导进行中 → 用户点击右上角 X → 界面渐变（500ms）回到对话界面 → AI 发送"好一点了吗？想继续聊可以继续，想安静也没关系。"
5. **I-01-05 暗色模式切换**：用户点击顶部栏太阳/月亮图标 → 全页 CSS 变量切换为暗色值（300ms 过渡）→ 图标旋转切换
6. **I-01-06 安全守护触发**：对话中 AI 检测到自伤/自杀信号 → AI 回复共情确认内容 → 对话流中在 AI 回复后渐入安全守护资源卡片（热线电话列表）→ 用户继续对话时不再重复弹出热线
7. **I-01-07 化石生成提示**：8 轮以上对话后用户发送结束信号（如"我要睡了"/ "晚安"）→ 5 秒后底部上浮化石预览卡 → 用户可点击查看 → 可点击"写一封回声信"→ 可点击关闭
8. **I-01-08 消息字数限制**：用户输入超过 1000 字 → 输入框下方出现柔和提示"消息太长了，分开说好不好，我不会走的" → 发送按钮禁用
9. **I-01-09 空白消息处理**：用户发送空白/仅空格消息 → AI 回应"嗯，在的。" → 显示极简确认
10. **I-01-10 消息流自动滚动**：新消息追加后 → 消息流自动滚动到底部 → 若用户已向上滚动查看历史消息，则不自动滚动，而是显示"有新消息"提示按钮
11. **I-01-11 纯 emoji/表情包消息**：用户发送纯 emoji 或表情包 → 消息正常追加到消息流 → AI 以匹配的情绪温度回应（开心 emoji → 轻松回应，悲伤 emoji → 温柔回应），不强行文字化解读
12. **I-01-12 语音录音时长限制**：录音进行中计时达到 120 秒 → 录音自动停止 → 显示"说了好多呢，先发这些吧" → 开始转写 → 转写结果填入输入框
13. **I-01-13 呼吸锚点减弱动画**：系统设置 prefers-reduced-motion 时 → 呼吸圆不做膨胀/收缩动画 → 改为柔和的色彩明暗渐变呼吸效果 → 文字引导节奏不变

### [P02] 情绪化石/日记页

**路由**：/fossils
**关联 PRD 功能**：3.5 情绪化石, 3.6 情绪地貌, 3.7 回声信, 3.10 历史对话与搜索
**页面定位**：情绪沉淀空间。用户在此回看自己的情绪旅程，化石流是核心，地貌和回声信是延伸。

#### 页面结构

- **区域 A - 顶部标题区**：页面标题"化石层"，右侧搜索图标按钮
- **区域 B - 子 Tab 切换**：「化石流 | 地貌 | 回声信 | 回忆」四个子 Tab，pill 形切换器
- **区域 C - 化石流视图（默认 Tab）**：纵向滚动的情绪化石卡片列表，按时间倒序。每块化石以其情绪色彩为左侧边条
- **区域 D - 地貌视图**：简化版情绪时间轴，横向滚动。水平轴为时间，纵轴为情绪起伏。每个化石是可点击的光点
- **区域 E - 回声信视图**：回声信列表，分为"待开启"和"已开启"两个分组
