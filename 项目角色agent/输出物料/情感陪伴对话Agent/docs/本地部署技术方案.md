# 留白 (Liminal) 本地部署技术方案

> 更新时间：2026-04-07
> 定位：单用户本地私有部署，双机分工架构

---

## 一、硬件配置

| 角色 | 设备 | 配置 |
|------|------|------|
| LLM 推理服务器 | Windows PC | RTX 5070 Ti 16GB VRAM / 64GB RAM |
| 应用服务器 | Mac Studio (2022) | Apple M1 Max / 32GB 统一内存 / macOS Sequoia |

---

## 二、整体架构

```
[电脑1 - Windows / LLM 服务器]
  llama-server (Gemma 26B)
  → 暴露 http://192.168.x.x:8080/v1  (OpenAI 兼容格式)

         ↕ 局域网 HTTP（< 1ms）

[电脑2 - Mac Studio / 应用服务器]
  Next.js 前端              :3000
  FastAPI Gateway           :8000
  ├── STT 服务 Qwen3-ASR    :8001
  ├── TTS 服务 IndexTTS-2   :8002
  └── 记忆库 API（独立部署） :????  （接口对接中）
```

**调用链路（单次对话）：**

```
用户语音输入
  → Qwen3-ASR (STT, Mac CPU/MPS, ~1s)
  → Gemma 26B (LLM, Windows GPU, ~2s，含记忆召回注入)
  → IndexTTS-2 (TTS, Mac MPS, ~0.5s，流式)
  → 播放语音
总感知延迟：~3.5s
```

## 三、LLM 服务（电脑1 · Windows）

### 模型选型

| 模块 | 模型文件 | 大小 |
|------|---------|------|
| 文本模型 | gemma-4-26B-A4B-it-UD-Q3_K_M.gguf | ~12.5 GB |
| 视觉模块（按需） | mmproj-F16.gguf | ~1.2 GB |

### 推理引擎

**llama-server（llama.cpp 原生，零额外开销）**

下载地址：[llama.cpp releases](https://github.com/ggerganov/llama.cpp/releases)
选择版本：`llama-xxx-bin-win-cuda-cu12.x.x-x64.zip`

### 启动脚本（start.bat）

```batch
@echo off
llama-server.exe ^
  -m "D:\models\gemma-4-26B-A4B-it-UD-Q3_K_M.gguf" ^
  --n-gpu-layers 999 ^
  --flash-attn ^
  --ctx-size 4096 ^
  --host 0.0.0.0 ^
  --port 8080 ^
  --log-disable
echo LLM 服务已启动：http://0.0.0.0:8080
pause
```

### 关键参数说明

| 参数 | 作用 |
|------|------|
| `--n-gpu-layers 999` | 全部层加载到 GPU |
| `--flash-attn` | KV Cache 节省 30-40% VRAM |
| `--ctx-size 4096` | 单用户 4K context 足够 |
| `--host 0.0.0.0` | 局域网可访问 |

### VRAM 分配（16GB）

```
Gemma 26B Q3_K_M 模型权重：~12.5 GB
KV Cache (4K ctx, flash-attn)：~2.5 GB
系统余量：~1 GB
```

## 四、STT 服务（电脑2 · Mac）

### 模型选型：Qwen3-ASR-1.7B

**选择理由：**
- 阿里 2026年1月发布，中文 ASR 当前最优
- 原生支持中文方言（粤语、四川话等）
- 噪音环境表现优于 Whisper large-v3
- 1.7B 参数，M1 Max MPS 加速 ~1s 完成

**安装：**
```bash
pip install qwen-asr
```

**服务接口（FastAPI 封装）：**
```
POST :8001/transcribe
Body: { audio: base64 }
Response: { text: "转录文字", duration: 0.98 }
```

## 五、TTS 服务（电脑2 · Mac）

### 模型选型：IndexTTS-2

**选择理由（对比 CosyVoice 2）：**

| 对比项 | CosyVoice 2 | IndexTTS-2 |
|--------|------------|------------|
| 情感控制方式 | 需要情感录音参考 | **纯文字描述驱动** |
| 中文质量 | 优秀 | **更优（汉字拼音混合建模）** |
| 训练数据 | — | 5.5万小时（3万中文）|
| 情感评测 | 基准 | **全面超越（人工盲测）** |
| 开源协议 | Apache 2.0 | Apache 2.0 |

**情感驱动示例（与 PRD 情绪体系完美契合）：**
```python
# AI 根据识别到的用户情绪，自动调整语音情感
tts.generate(
    text="你说没事，但好像有些话没说出来...",
    emotion="用温柔低沉的声音，带一点心疼"
)
```

**安装：**
```bash
git clone https://github.com/index-tts/index-tts
```

**服务接口：**
```
POST :8002/synthesize
Body: { text: "文字内容", emotion: "情感描述" }
Response: audio/wav 流式返回
```

**Mac 资源占用：** ~3GB 统一内存，MPS 加速，首帧 ~500ms

## 六、记忆库（独立部署，接口对接中）

### 记忆分层设计（对应 PRD 3.4）

| 层级 | 内容 | 存储方式 | 检索方式 |
|------|------|---------|---------|
| 即时记忆 | 当次对话上下文 | Context Window | 直接注入 |
| 事件记忆 | 具体事件、人物、时间 | 向量库 | RAG 召回 |
| 情绪记忆 | 情绪触发模式（周度聚合）| 向量库 | RAG 召回 |
| 人格理解 | 性格、价值观（月度聚合）| 结构化存储 | Prompt 注入 |

### 推荐技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| Embedding | BGE-M3 (~570MB) | 中文最优，sparse+dense 双路 |
| Reranker | BGE-Reranker-v2-M3 (~570MB) | 与 BGE-M3 同系列，配套最佳 |
| 向量数据库 | Qdrant | 轻量单二进制，性能最优 |

### RAG 检索流程

```
用户输入
  → BGE-M3 向量化
  → Qdrant 粗召回 Top 20
  → BGE-Reranker-v2-M3 精排 Top 5
  → 注入 Gemma 26B context window
```

### 待对接接口（预留）

```
POST /memory/write   — 对话结束后写入事件/情绪记忆
POST /memory/search  — 对话前召回相关记忆
PUT  /memory/update  — 用户纠正记错内容
DELETE /memory/:id   — 用户主动删除记忆
```
