# A股分红股排行榜 设计色彩报告

> 调研时间：2026-03-24
> 目标用户画像：个人投资者（散户为主），25-55 岁，关注稳定分红收入的中长期投资者
> 产品调性关键词：专业可信、简洁清晰、数据驱动、现代感

## 1. 行业色彩语言

### 1.1 行业色彩惯例

金融数据类产品有明确的色彩传统：

- **蓝色系是绝对主流**：蓝色传递"信赖、稳定、专业"，是金融行业约定俗成的安全色。从银行（工行蓝、建行蓝）到券商（中信蓝）到数据平台（集思录蓝），蓝色渗透率极高。 -- 来源：[Phoenix Strategy Group](https://www.phoenixstrategy.group/blog/best-color-palettes-for-financial-dashboards)
- **红绿用于涨跌编码**：中国市场红涨绿跌（与欧美相反），这是不可更改的文化惯例。分红类数据中，红色还有"分红=利好"的正面联想。
- **低饱和度是大势**：现代金融 UI 趋向低饱和度配色（柔和蓝、淡灰、雾蓝），降低用户对财务工具的焦虑感。 -- 来源：[即时设计](https://js.design/special/article/financial-app-interface-design.html)
- **金色/琥珀色作为高级感点缀**：红利/分红类产品中，金色暗示"财富、收益"，可作为强调色使用。

### 1.2 头部产品用色统计

| 产品 | 主色 | 辅色 | 强调色 | 整体风格 |
|------|------|------|--------|---------|
| 东方财富 | #E4393C（正红） | #FFFFFF（白底） | #FF8C00（橙色） | 门户风、信息密集、红色主导 |
| 集思录 | #006798（深蓝） | #F5F5F5（浅灰底） | #FFC107（黄色高亮） | 数据密集、蓝色专业、温度编码 |
| 乐咕乐股 | #337AB7（中蓝） | #FFFFFF（白底） | #333333（深灰文字） | 极简功能、白底蓝链接、朴素 |
| 同花顺 | #E63E30（中国红） | #2B3E50（深蓝灰） | #FF9900（橙色） | 国风红色、黑底行情、专业感 |
| 乌龟量化 | #4A90D9（柔蓝） | #FFFFFF（白底） | #333333（深灰文字） | 极简白底、功能优先、无装饰 |

**统计结论**：
- 蓝色系占比 3/5（集思录、乐咕乐股、乌龟量化）
- 红色系占比 2/5（东方财富、同花顺），但这两者是综合金融平台，红色服务于"行情涨跌"语境
- 对于垂直数据工具，蓝色系更合适
- 所有竞品均未使用暗色模式作为主界面

### 1.3 色彩心理学参考

**目标用户群体**（25-55 岁中国散户投资者）的色彩偏好：
- **蓝色**：信赖、冷静、专业 -- 最安全的选择，符合用户对金融工具的心理预期
- **绿色**：在 A 股语境中是"下跌色"，需谨慎使用 -- 建议仅用于"成功/确认"状态，避免大面积使用
- **红色**：在 A 股语境中是"上涨色"和"利好色" -- 分红场景有正面联想，但大面积使用会显得激进
- **金色/琥珀**：财富、收益、高级感 -- 适合作为分红类产品的强调色
- **文化禁忌**：避免大面积纯黑配白底文字（丧葬联想）；避免过多绿色（A 股下跌色）

## 2. 推荐配色方案（3 套）

### 方案 A："信赖蓝"（Trust Blue）

- **调性关键词**：专业 / 可信赖 / 冷静 / 数据驱动
- **适合场景**：金融数据工具、排行榜、分析仪表板
- **色彩体系**：
  - 主色：#1A73E8（明蓝） -- 用途：品牌色/主按钮/Tab 激活态/链接
  - 辅色：#5F6368（中灰） -- 用途：次要文字/辅助操作/边框
  - 强调色：#F59E0B（琥珀金） -- 用途：分红高亮/评分星标/CTA/重点数据
  - 成功色：#34A853（绿色） -- 用途：数据更新成功/正向指标
  - 警告色：#FBBC04（明黄） -- 用途：数据过期提示/注意标记
  - 错误色：#EA4335（谷歌红） -- 用途：错误提示/数据异常
  - 背景色（亮）：#F8FAFC -- 用途：页面主背景
  - 背景色（暗）：#1E293B -- 用途：暗色模式主背景（如支持）
  - 文字主色：#1F2937 -- 用途：标题/正文
  - 文字次色：#6B7280 -- 用途：说明文字/次要信息
- **对比度检查**：主色 #1A73E8 在白底上对比度 4.6:1（达到 WCAG AA）；文字主色 #1F2937 在 #F8FAFC 上对比度 13.8:1（达到 AAA）
- **参考来源**：Google Material Design 蓝色系 + Dribbble 金融仪表板设计
- **使用建议**：60% 白/浅灰背景 + 30% 蓝色系（导航/表头/按钮）+ 10% 琥珀金（高亮/CTA）

### 方案 B："沉稳深蓝"（Corporate Navy）

- **调性关键词**：权威 / 沉稳 / 高端 / 传统金融
- **适合场景**：面向成熟投资者的专业工具
- **色彩体系**：
  - 主色：#003366（深海蓝） -- 用途：导航栏/品牌标识/主标题
  - 辅色：#00509E（中蓝） -- 用途：链接/次要按钮/图表线条
  - 强调色：#D4A843（哑光金） -- 用途：分红数据高亮/排名数字/评分
  - 成功色：#2E7D32（森林绿） -- 用途：正向数据
  - 警告色：#F9A825（深黄） -- 用途：警告状态
  - 错误色：#C62828（深红） -- 用途：错误状态
  - 背景色（亮）：#F5F7FA -- 用途：页面主背景
  - 背景色（暗）：#0F172A -- 用途：暗色模式
  - 文字主色：#1A1A2E -- 用途：标题/正文
  - 文字次色：#64748B -- 用途：说明文字
- **对比度检查**：主色 #003366 在白底上对比度 10.5:1（AAA 级）；强调色 #D4A843 在深蓝 #003366 上对比度 5.2:1（AA 级）
- **参考来源**：传统银行/券商视觉系统 + Phoenix Strategy Group 金融仪表板配色指南
- **使用建议**：60% 浅灰白背景 + 25% 深蓝系（导航/表头）+ 15% 金色系（数据高亮）

### 方案 C："清新科技"（Modern Teal）

- **调性关键词**：现代 / 清新 / 创新 / 年轻化
- **适合场景**：面向年轻投资者的新锐数据工具
- **色彩体系**：
  - 主色：#0D9488（翠绿青） -- 用途：品牌色/主按钮/导航激活
  - 辅色：#6366F1（靛蓝紫） -- 用途：图表/次要操作/装饰
  - 强调色：#F97316（活力橙） -- 用途：CTA/高亮/分红数据标注
  - 成功色：#22C55E（明绿） -- 用途：正向数据
  - 警告色：#EAB308（亮黄） -- 用途：警告状态
  - 错误色：#EF4444（亮红） -- 用途：错误状态
  - 背景色（亮）：#F0FDFA -- 用途：主背景（极淡青）
  - 背景色（暗）：#134E4A -- 用途：暗色模式
  - 文字主色：#0F172A -- 用途：标题/正文
  - 文字次色：#64748B -- 用途：说明文字
- **对比度检查**：主色 #0D9488 在白底上对比度 3.8:1（需加大字号或加粗才达 AA）；文字主色 #0F172A 在浅青底上对比度 14.2:1（AAA）
- **参考来源**：Tailwind CSS Teal 色系 + Dribbble 金融科技设计趋势
- **使用建议**：60% 白/浅青背景 + 25% 青色系 + 15% 橙色/紫色点缀
- **注意**：青色在 A 股领域偏非传统，可能降低部分保守用户的信任感

|------|-------------|---------------|---------------|

| 差异化程度 | ★★★☆☆ | ★★☆☆☆ | ★★★★★ |
| 无障碍友好度 | ★★★★★ | ★★★★★ | ★★★★☆ |
| 目标用户匹配 | ★★★★★ | ★★★★☆ | ★★★☆☆ |

**首选推荐**：方案 A "信赖蓝"。理由：
1. 蓝色是金融行业最安全的选择，用户心理预期匹配度最高
2. 琥珀金强调色与"分红/收益"主题天然契合，形成视觉记忆点
3. 明蓝 #1A73E8 比竞品的蓝色更鲜活现代，既专业又不沉闷
4. 对比度全部达到 WCAG AA，无障碍友好
5. 在竞品普遍使用"暗淡蓝"的市场中，明蓝+琥珀金组合足够差异化

## 3. 字体趋势

### 3.1 行业字体风格

- **金融数据类产品以无衬线字体为绝对主流**：无衬线字体传递现代感和可读性，尤其在数据密集的表格场景中
- 头部竞品普遍使用系统默认字体（微软雅黑/苹方），较少使用自定义字体
- 数字展示场景建议使用等宽或 Tabular 数字字体，确保数据列对齐
- 英文数字部分，Inter 和 DIN 系列在金融 UI 中非常流行 -- 来源：[金融 UI 设计趋势](https://js.design/special/article/financial-app-interface-design.html)

### 3.2 推荐字体搭配

| 用途 | 推荐字体 | 备选 | 说明 |
|------|---------|------|------|
| 英文标题 | Inter Bold | Poppins SemiBold | Inter 在数据界面中可读性极佳，免费商用（SIL OFL） |
| 英文正文/数字 | Inter Regular | DM Sans Regular | 同上，数字字形设计对金融数据友好 |
| 中文标题 | Noto Sans SC Bold（思源黑体 Bold） | 阿里巴巴普惠体 Bold | 免费商用，字重丰富（7 级），与 Inter 搭配协调 |
| 中文正文 | Noto Sans SC Regular | 阿里巴巴普惠体 Regular | 免费商用，可读性好 |
| 代码/等宽 | JetBrains Mono | Fira Code | 股票代码等需等宽显示的场景 |

**字体加载策略建议**：使用 Google Fonts CDN 加载 Inter + Noto Sans SC，设置 `font-display: swap` 避免 FOIT。

## 4. 视觉风格趋势

### 4.1 行业视觉趋势

根据 2025-2026 年 UI 设计趋势调研：

- **布局**：
  - Bento Grid（便当盒布局）在数据仪表板中持续流行，适合将不同维度的排行榜模块化展示 -- 来源：[2025 UI 趋势](https://www.lizhiruanjian.com/post/01HZ3HVPT1P5VKADJ1ME2JYZBY)
  - 卡片式布局替代传统表格，提升视觉层次和呼吸感
  - 单栏居中布局在轻量工具类产品中回归

- **视觉风格**：
  - 低饱和度 + 柔和阴影是当前金融产品主流
  - 玻璃拟态（Glassmorphism）在金融应用中持续流行，但需克制使用
  - 微渐变替代纯色块，增加层次感
  - 来源：[CoDesign 腾讯](https://codesign.qq.com/hc/article/2025-design-trends/)

- **暗色/亮色**：
  - 亮色模式仍是金融数据工具的默认选择（数据可读性优先）
  - 暗色模式作为可选项提供，不做默认

- **动效**：
  - 微交互（hover 变色、数据切换淡入淡出）替代复杂动画
  - 数据加载使用骨架屏（Skeleton Screen）而非传统 loading 图标
  - 排名变化使用数字滚动动效（CountUp.js 风格）

- **圆角**：
  - 中等圆角（8-12px）是金融产品的安全选择，既现代又不失专业
  - 避免超大圆角（20px+），会降低专业感

### 4.2 风格方向建议

- **推荐风格**：现代简洁（Clean & Modern） -- 适合金融数据排行榜，竞品普遍偏老旧，现代设计即差异化
- **暗色/亮色**：亮色模式为主（白/浅灰背景），暗色模式作为可选 -- 金融数据需要高可读性，亮色更适合
- **圆角策略**：8px（卡片/按钮）/ 12px（大卡片/弹窗）/ 6px（输入框/小组件） -- 中等圆角兼顾现代感和专业感

## 5. 风格参考案例（文字情绪板）

### 案例 1：Google Finance（金融数据简洁典范）

- **整体感受**：干净、信赖、高效、无噪音
- **配色特点**：白底 + 蓝色主调 + 绿红涨跌编码，色彩克制，留白充足
- **排版特点**：大号数字突出关键指标，正文 14-16px，行间距 1.5-1.6，层级清晰
- **组件风格**：圆角卡片带微阴影，无边框表格，极简按钮
- **动效特点**：数字变化有平滑过渡，Tab 切换有淡入淡出，整体克制
- **值得借鉴**：数据展示的克制和层次感，白色空间的运用，数字排版的精致
- **需要规避**：过度简化导致功能不足（Google Finance 缺乏高级筛选）

### 案例 2：TradingView（数据密集但不混乱的典范）

- **整体感受**：专业、现代、深度、可定制
- **配色特点**：暗色主题 + 蓝色强调 + 绿红涨跌，暗色背景降低视觉疲劳
- **排版特点**：紧凑排版，字号层级丰富（6-7 级），数据密度高但不凌乱
- **组件风格**：深色卡片 + 微边框 + hover 高亮，Screener 表格支持列拖拽和自定义
- **动效特点**：实时数据闪烁更新，平滑的图表绘制动画
- **值得借鉴**：高信息密度下的视觉组织能力，筛选器的交互模式
- **需要规避**：学习曲线陡峭，不适合我们的"简洁"定位；暗色主题不是我们的首选

### 案例 3：集思录新版界面（国内金融数据产品参考）

- **整体感受**：功能性强、数据丰富、略显拥挤
- **配色特点**：蓝色主调 #006798 + 白/灰底 + 黄色高亮数据区，温度编码（青→红）
- **排版特点**：表格主导，12-14px 正文，行间距偏紧，信息密度高
- **组件风格**：标准表格 + 下拉筛选器 + 标签式导航
- **动效特点**：几乎无动效，纯静态页面
- **值得借鉴**：筛选维度的丰富度（12 个维度），数据列的选择（28 列可选）
- **需要规避**：信息过载问题，视觉层次不够清晰，动效缺失导致体验生硬

### 案例 4：Stripe Dashboard（非金融数据，但数据仪表板设计标杆）

- **整体感受**：精致、现代、克制、高级
- **配色特点**：白底 + 蓝紫主调（#635BFF）+ 极少量强调色，色彩高度统一
- **排版特点**：大标题 + 充足留白 + 数据区紧凑，层级通过字重和颜色区分
- **组件风格**：圆角卡片（12px）+ 多层柔和阴影 + 微妙的 hover 状态
- **动效特点**：页面切换有平滑过渡，数据加载用骨架屏，hover 效果细腻
- **值得借鉴**：设计系统的一致性，阴影和圆角的精致处理，留白节奏
- **需要规避**：过于"科技公司"的感觉，需要调整为更贴合中国金融用户的调性

## 6. 设计红线

- **必须避免**：
  - 不要使用大面积绿色作为正面含义（A 股绿=跌，会引起反感）
  - 不要使用过多花哨动画（金融场景用户追求效率，非娱乐）
  - 不要使用衬线字体作为主字体（数据密集场景可读性差）
  - 不要在排行榜中使用 emoji 图标（降低专业感）

- **色彩禁忌**：
  - 避免纯黑 #000000 背景 + 白色文字的高对比方案（刺眼、非金融调性）
  - 避免大面积使用 #00FF00 等高饱和绿色（霓虹感，不专业）
  - 避免红绿色彩编码与 A 股涨跌惯例冲突（红=涨/正面，绿=跌/负面）
  - 避免紫色/粉色作为主色调（与金融行业调性不符）

- **风格陷阱**：
  - 不要盲目追求暗色模式作为默认（亮色模式在数据阅读场景中更优）
  - 不要过度使用玻璃拟态效果（金融产品需要清晰边界，模糊会降低数据可读性）
  - 不要为了"差异化"而使用非常规配色（金融用户对色彩有保守预期，过于跳脱会降低信任感）
  - 不要忽视表格场景的排版细节（数字对齐、千分位、小数位一致性）
