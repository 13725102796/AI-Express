# 分红股排行榜 产品需求文档（PRD）

## 1. 产品概述

- **产品名称**：分红股排行榜
- **一句话定位**：A 股分红数据的极简排行榜工具，用三个维度帮投资者快速找到最值得关注的分红股
- **目标用户**：关注股息收入的 A 股个人投资者（散户），尤其是追求稳定分红的中长期持有者
- **核心价值**：提供竞品均未覆盖的"连续分红年数"排名和"综合评分"排名，用极简界面降低分红选股的认知成本

## 2. 用户画像

| 维度 | 描述 |
|------|------|
| 角色 | 25-55 岁的 A 股散户投资者，关注分红收益，偏好中长期持有策略 |
| 痛点 | 1. 现有工具没有"连续分红年数"排名，需手动统计；2. 综合平台信息过载，找分红数据需多次跳转；3. 没有综合评分体系，无法一目了然地对比分红质量 |
| 期望 | 打开网页就能看到三个维度的分红排行榜（稳定性/金额/综合），数据清晰可信，操作简单 |
| 使用场景 | 1. 周末选股研究时，快速筛选高分红标的；2. 年报季查看最新分红数据排名变化；3. 手机端碎片时间浏览分红排行 |

## 3. 功能需求

### 3.1 排行榜 Tab 切换

- **描述**：页面顶部提供三个排行榜 Tab，用户点击切换查看不同维度的排名
- **用户故事**：作为投资者，我希望在同一页面上通过 Tab 切换查看不同维度的分红排名，以便快速对比不同角度的分红数据
- **验收标准**：
  - Given 用户打开网页
  - When 页面加载完成
  - Then 默认展示"综合排行"Tab 的数据，Tab 栏同时显示"稳定分红"、"分红最多"、"综合排行"三个选项
  - Given 用户点击"稳定分红"Tab
  - When Tab 切换完成
  - Then 表格数据按连续分红年数从高到低排序，当前 Tab 有视觉激活状态，切换耗时 < 200ms
- **优先级**：P0
- **边界条件与异常场景**：
  - 当两只股票连续分红年数相同时，按股息率降序作为次要排序条件
  - 当两只股票累计分红金额相同时，按股息率降序作为次要排序条件
  - Tab 切换过程中禁止重复点击（防抖处理，间隔 300ms）

### 3.2 稳定分红排行榜

- **描述**：按连续分红年数从高到低排序，展示 A 股中分红最稳定的公司
- **用户故事**：作为关注分红稳定性的投资者，我希望看到按连续分红年数排序的排行榜，以便找到长期稳定分红的"养老股"
- **验收标准**：
  - Given 用户选中"稳定分红"Tab
  - When 数据展示完成
  - Then 表格包含以下列：排名、股票代码、股票名称、所属行业、连续分红年数、最近年度股息率(%)、最近年度每股分红(元)
  - Given 数据已加载
  - When 用户查看排名
  - Then 排名按连续分红年数降序排列，连续分红年数相同时按股息率降序
- **优先级**：P0
- **边界条件与异常场景**：
  - "连续分红年数"定义：从最近一个分红年度起向前追溯，连续每年均有现金分红记录的年数。中间有任意一年未分红则中断计数
  - 仅统计现金分红（不含送股、转增）
  - 最低展示条件：连续分红年数 >= 1 年

### 3.3 分红最多排行榜

- **描述**：按累计分红金额和股息率综合展示分红规模最大的公司
- **用户故事**：作为关注分红收益的投资者，我希望看到分红金额和股息率最高的公司排名，以便找到高收益的分红股
- **验收标准**：
  - Given 用户选中"分红最多"Tab
  - When 数据展示完成
  - Then 表格包含以下列：排名、股票代码、股票名称、所属行业、最近年度股息率(%)、近 3 年平均股息率(%)、最近年度每股分红(元)、累计分红总额(亿元)
  - Given 数据已加载
  - When 用户查看排名
  - Then 默认按最近年度股息率降序排列
- **优先级**：P0
- **边界条件与异常场景**：
  - 股息率 = 最近年度每股现金分红 / 当前股价 * 100%
  - 近 3 年平均股息率 = 近 3 年每股现金分红之和 / 3 / 当前股价 * 100%
  - 累计分红总额包含上市以来所有现金分红
  - 当股价数据不可用时，股息率列显示"--"

### 3.4 综合排行榜

- **描述**：基于分红稳定性和分红金额的加权评分，给出综合分红质量排名
- **用户故事**：作为不知道该看哪个维度的投资者，我希望有一个综合评分告诉我哪些股票的分红质量最好，以便做出投资参考
- **验收标准**：
  - Given 用户选中"综合排行"Tab（默认 Tab）
  - When 数据展示完成
  - Then 表格包含以下列：排名、股票代码、股票名称、所属行业、综合评分、连续分红年数、最近年度股息率(%)、近 3 年平均股息率(%)
  - Given 数据已加载
  - When 用户查看排名
  - Then 排名按综合评分降序排列
- **优先级**：P0
- **边界条件与异常场景**：
  - 综合评分算法：稳定性得分（连续分红年数归一化，占 40%）+ 股息率得分（近 3 年平均股息率归一化，占 35%）+ 分红规模得分（累计分红总额归一化，占 25%）。总分 0-100 分
  - 归一化方法：(当前值 - 最小值) / (最大值 - 最小值) * 100
  - 综合评分保留 1 位小数
  - 评分公式应在页面底部"评分说明"区域公开展示，确保透明度

### 3.5 手动数据更新

- **描述**：页面提供"更新数据"按钮，用户点击后触发数据抓取和更新
- **用户故事**：作为用户，我希望通过点击按钮手动触发数据更新，以便在需要时获取最新的分红数据
- **验收标准**：
  - Given 用户看到页面顶部的"更新数据"按钮和上次更新时间
  - When 用户点击"更新数据"按钮
  - Then 按钮变为加载状态（显示旋转图标 + "更新中..."文字），禁止重复点击
  - Given 数据更新请求已发送
  - When 更新完成（成功）
  - Then 按钮恢复为可点击状态，"上次更新时间"刷新为当前时间（格式：YYYY-MM-DD HH:mm），排行榜数据刷新
  - Given 数据更新请求已发送
  - When 更新失败
  - Then 按钮恢复为可点击状态，显示错误提示"数据更新失败，请稍后重试"（红色文字，5 秒后自动消失）
- **优先级**：P0
- **边界条件与异常场景**：
  - 更新过程中用户切换 Tab，更新不中断，完成后自动刷新当前 Tab 数据
  - 更新超时时间：60 秒。超时后显示"更新超时，请稍后重试"
  - 连续点击更新的冷却时间：同一用户 5 分钟内只能触发 1 次更新
  - 上次更新时间持久化存储（localStorage），页面刷新后仍可显示

### 3.6 排行榜数据表格

- **描述**：使用表格形式展示排行榜数据，支持基础的排序和分页
- **用户故事**：作为用户，我希望在整洁的表格中查看排行榜数据，并能翻页查看更多数据
- **验收标准**：
  - Given 排行榜数据已加载
  - When 表格渲染完成
  - Then 每页展示 50 条数据，表头固定，表格支持竖向滚动
  - Given 用户查看表格
  - When 数据超过 50 条
  - Then 表格底部显示分页器（上一页/下一页/页码），总条数显示
  - Given 用户查看表格
  - When 鼠标 hover 某一行
  - Then 该行背景色变为浅蓝高亮，区分当前焦点行
- **优先级**：P0
- **边界条件与异常场景**：
  - 数值列右对齐，文字列左对齐
  - 股息率/评分等浮点数统一保留 2 位小数
  - 金额超过万元的使用"万"为单位，超过亿元的使用"亿"为单位
  - 股票代码格式：6 位数字（如 600519）
  - 空数据显示"--"，不显示 0 或 null

### 3.7 页面顶部信息区

- **描述**：页面顶部展示产品标题、数据概览和更新控制
- **用户故事**：作为用户，我希望打开页面就能看到数据的基本概况和最后更新时间，以便判断数据的时效性
- **验收标准**：
  - Given 用户打开页面
  - When 页面加载完成
  - Then 顶部区域展示：产品标题"分红股排行榜"、数据概览（共 N 只分红股、数据更新时间）、"更新数据"按钮
- **优先级**：P0
- **边界条件与异常场景**：
  - 首次访问无数据时，显示"暂无数据，请点击'更新数据'获取最新排行"

### 3.8 页面底部信息区

- **描述**：页面底部展示免责声明、评分说明和数据来源
- **用户故事**：作为用户，我希望了解数据来源和评分逻辑，以便建立对数据的信任
- **验收标准**：
  - Given 用户滚动到页面底部
  - When 底部区域可见
  - Then 展示三个信息模块：(1) 评分说明（综合评分算法的公开说明）；(2) 数据来源（注明数据来自公开市场数据）；(3) 免责声明（"本站数据仅供参考，不构成投资建议"）
- **优先级**：P1
- **边界条件与异常场景**：
  - 免责声明文字使用灰色小字号，不影响主要内容阅读

## 4. 非功能需求

- **性能要求**：
  - 页面首次加载时间 < 3 秒（有缓存数据时 < 1 秒）
  - Tab 切换响应时间 < 200ms
  - 数据更新操作超时时间：60 秒
  - 表格滚动流畅，无卡顿（目标 60fps）

- **安全要求**：
  - 数据更新接口需防止恶意频繁调用（频率限制）
  - 不存储任何用户个人信息
  - HTTPS 部署

- **兼容性要求**：
  - 浏览器：Chrome 90+、Safari 14+、Firefox 88+、Edge 90+
  - 设备：桌面端（1280px+）和移动端（375px+）均可正常使用
  - 响应式断点：375px / 768px / 1280px

- **可访问性要求**：
  - 文字与背景的对比度达到 WCAG AA 标准（4.5:1）
  - 表格有适当的 aria-label 和表头标注
  - 支持键盘导航（Tab 切换、分页操作）

## 5. 信息架构

### 页面结构（单页应用）

```
分红股排行榜
├── 顶部信息区
│   ├── 产品标题 + Logo
│   ├── 数据概览（分红股总数 + 最后更新时间）
│   └── "更新数据"按钮
├── 排行榜区域
│   ├── Tab 栏（稳定分红 | 分红最多 | 综合排行）
│   └── 数据表格（含分页）
└── 底部信息区
    ├── 评分说明
    ├── 数据来源
    └── 免责声明
```

### 导航逻辑

- 单页应用，无多页跳转
- Tab 切换为核心导航方式
- 分页为辅助导航

### 核心用户动线

1. 用户打开页面 -> 看到综合排行榜（默认 Tab）
2. 用户点击"更新数据"获取最新数据（首次访问必须操作）
3. 用户切换 Tab 查看不同维度的排名
4. 用户翻页查看更多数据
5. 用户在底部查看评分说明和免责声明

## 6. 设计指引

- **风格方向**：现代简洁（Clean & Modern），与竞品的陈旧 UI 形成鲜明差异。数据优先，减少视觉噪音。参考 Google Finance 的克制感 + Stripe Dashboard 的精致感
- **参考产品**：
  - Google Finance：数据展示的克制和留白
  - Stripe Dashboard：卡片和阴影的精致处理
  - 集思录：筛选维度的功能参考（UI 不参考）
- **品牌调性关键词**：专业可信、简洁高效、数据驱动、现代感、有呼吸感
- **色彩方向**：蓝色系主调（金融行业信赖色）+ 琥珀金/暖色作为分红数据的强调色，传递"收益、财富"语义
- **排版方向**：无衬线字体，数字需等宽对齐，表格间距舒适不拥挤
- **关键设计约束**：
  - 不使用 emoji 作为 UI 图标
  - 红色 = A 股涨/利好，绿色 = A 股跌/负面（中国市场惯例）
  - 表格数字右对齐，文字左对齐
  - 移动端表格需可横向滚动或自适应简化列

## 7. 范围边界

- **MVP 包含**：
  - 三大排行榜（稳定分红/分红最多/综合排行）及 Tab 切换
  - 手动触发数据更新 + 更新时间戳
  - 排行榜数据表格（含分页）
  - 页面顶部信息区（标题/概览/更新按钮）
  - 页面底部信息区（评分说明/数据来源/免责声明）
  - 响应式设计（桌面端 + 移动端）

- **MVP 不包含（后续迭代）**：
  - 行业/板块筛选功能（P1，第二迭代）
  - 个股分红详情弹窗/卡片（P1，第二迭代）
  - 分红趋势可视化图表（P2，第三迭代）
  - 数据导出功能（P2）
  - 暗色模式（P2）
  - 用户账号和收藏功能（P2）

- **明确排除**：
  - 社区/评论功能：不符合"轻量工具"定位
  - 自动定时更新：用户明确要求手动触发
  - 实时行情数据：超出分红排行榜的范围
  - 个股交易/买卖功能：不涉及交易
  - 投资建议/荐股内容：合规红线

## 8. 开放问题

1. **数据源选择**：AKShare 和 Tushare 均可获取分红数据，具体使用哪个库需要技术评估。建议优先 AKShare（完全免费），Tushare 作为备选
2. **综合评分权重**：当前设定为稳定性 40% + 股息率 35% + 分红规模 25%，此权重是否需要用户自定义？MVP 阶段暂不支持自定义，后续可考虑
3. **数据覆盖范围**：是否包含 ST 股？建议 MVP 阶段默认排除 ST 股（在底部说明中注明）
4. **分红数据的时间范围**：历史分红数据追溯多少年？建议追溯至公司上市以来的全部记录
5. **部署方案**：前后端分离还是全栈单体？需要技术架构确认
