# 个人知识库产品 - 设计色彩报告

> 调研时间：2026-03-20
> 调研范围：PKM/知识管理产品设计趋势

## 1. 行业色彩语言

### 1.1 头部竞品用色统计

| 产品 | 主色 | 辅色 | 风格关键词 |
|------|------|------|-----------|
| **Notion** | 黑 `#000000` + 白 `#FFFFFF` | 10 色系统（蓝 `#487CA5`、橙 `#D9730D`、绿 `#448361` 等） | 极简、克制、黑白为主、内容优先 |
| **Obsidian** | 深紫 `#7C3AED` + 深灰 `#1E1E2E` | 暗色主题为主，支持 Nord `#88C0D0`、Solarized `#268BD2` 等主题 | 暗色系、技术感、可高度定制 |
| **Mem.ai** | 白色背景 `#FFFFFF` + 黑色文字 `#1A1A1A` | 蓝紫渐变 `#6366F1` → `#8B5CF6` | 明亮简洁、AI 感、现代 |
| **Readwise** | 暖黄 `#F5A623` + 白 `#FFFFFF` | 深灰 `#37352F`、米色 `#FBF8F4` | 温暖、书卷气、阅读友好 |
| **Heptabase** | 蓝色 `#3B82F6` + 白 `#FFFFFF` | 浅灰 `#F5F5F5`、多彩卡片色系 | 清新、空间感、几何化、白板感 |

### 1.2 行业色彩趋势总结

- **极简中性为底**：几乎所有 PKM 产品都以黑/白/灰为基底色，内容区域保持中性
- **功能色克制使用**：主色只在按钮、链接、高亮等交互元素上出现
- **暗色模式成为标配**：所有主流产品都支持 Light/Dark 双主题
- **渐变回归**：Mem.ai 等新产品使用微妙渐变传达 AI/科技感
- **暖色系差异化**：Readwise 用暖黄色成功区分于一众「蓝色系」产品

### 1.3 色彩心理学参考

| 色彩方向 | 心理学含义 | 适用场景 |
|---------|-----------|---------|
| 蓝色系 | 信任、专业、理性、冷静 | 企业知识管理、专业工具 |
| 绿色系 | 成长、自然、平衡、知识 | 学习类产品、个人成长 |
| 紫色系 | 智慧、创造、高端、AI 感 | AI 驱动产品、创意工具 |
| 暖黄/橙色 | 温暖、活力、灵感、可及性 | 阅读类产品、个人使用 |
| 中性灰 | 专注、内容优先、无干扰 | 写作/笔记工具 |

## 2. 推荐配色方案

### 方案一：「深邃智慧」——蓝紫 AI 知识感

**设计理念**：以深蓝紫为主色，传达 AI 智能与知识深度，适合强调「AI 驱动」定位的产品。

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色（Primary） | `#4F46E5`（Indigo 600） | 按钮、链接、主要交互元素 |
| 主色-浅（Primary Light） | `#818CF8`（Indigo 400） | Hover 状态、次要强调 |
| 主色-深（Primary Dark） | `#3730A3`（Indigo 800） | 深色模式下的主色 |
| 辅色（Secondary） | `#06B6D4`（Cyan 500） | 标签、Badge、辅助元素 |
| 强调色（Accent） | `#F59E0B`（Amber 500） | 高亮标记、收藏、重要提示 |
| 成功色（Success） | `#10B981`（Emerald 500） | 成功状态、确认操作 |
| 警告色（Warning） | `#F59E0B`（Amber 500） | 警告提示 |
| 错误色（Error） | `#EF4444`（Red 500） | 错误状态、删除操作 |
| 背景-亮（BG Light） | `#FAFAFA` | 亮色模式主背景 |
| 背景-亮-次级 | `#F3F4F6`（Gray 100） | 卡片、侧边栏背景 |
| 背景-暗（BG Dark） | `#111827`（Gray 900） | 暗色模式主背景 |
| 背景-暗-次级 | `#1F2937`（Gray 800） | 暗色模式卡片背景 |
| 文字-主要（亮色） | `#111827`（Gray 900） | 标题、正文 |
| 文字-次要（亮色） | `#6B7280`（Gray 500） | 辅助文字、时间戳 |
| 文字-主要（暗色） | `#F9FAFB`（Gray 50） | 暗色模式标题、正文 |
| 文字-次要（暗色） | `#9CA3AF`（Gray 400） | 暗色模式辅助文字 |

**对比度检查**：
- `#4F46E5` on `#FAFAFA` → 对比度 6.3:1 (AA pass, AAA pass for large text)
- `#111827` on `#FAFAFA` → 对比度 17.4:1 (AAA pass)
- `#F9FAFB` on `#111827` → 对比度 17.2:1 (AAA pass)

**60-30-10 法则**：60% 中性背景（`#FAFAFA`/`#F3F4F6`）→ 30% 文字与结构色（`#111827`/`#6B7280`）→ 10% 主色与强调色（`#4F46E5`/`#F59E0B`）

## 3. 字体趋势

### 3.1 行业字体风格

- **无衬线体主导**：几乎所有 PKM/效率工具都使用无衬线体（sans-serif）
- **Variable Font 成为标配**：可变字体支持无级调节粗细，提升性能和灵活性
- **大 x-height**：提高屏幕可读性，Inter、SF Pro 等字体的设计方向
- **跨平台一致性**：选择在 Windows/macOS/iOS/Android 都能良好渲染的字体

### 3.2 推荐字体搭配

#### 英文字体

| 用途 | 首选 | 备选 | 说明 |
|------|------|------|------|
| **UI 界面 / 正文** | Inter | SF Pro（Apple 设备） | Inter 为 UI 设计量身定制，开源免费，Variable Font，大 x-height |
| **标题** | Inter 700 / Inter 800 | Mona Sans（GitHub 开源） | 同一字族不同字重，保持统一感 |
| **代码/等宽** | JetBrains Mono | Fira Code | 开源、支持连字（Ligature）、专为开发者设计 |
| **备选正文** | Figtree | DM Sans | 2025 新锐选择，Figtree 几何感更强，DM Sans 更经典 |

#### 中文字体

| 用途 | 首选 | 备选 | 说明 |
|------|------|------|------|
| **正文** | 思源黑体（Noto Sans SC） | 苹方（PingFang SC） | 思源黑体：Adobe+Google 开源，7 种字重，免费商用；苹方：Apple 设备原生 |
| **标题** | 思源黑体 Bold / Heavy | HarmonyOS Sans | 同字族加粗，或使用华为开源字体 |
| **备选** | 阿里巴巴普惠体 | OPPO Sans | 均为免费商用，风格现代 |

#### 推荐字体栈（CSS）

```css
/* 正文字体栈 */
font-family: 'Inter', -apple-system, BlinkMacSystemFont,
  'Noto Sans SC', 'PingFang SC', 'Microsoft YaHei',
  sans-serif;

/* 等宽字体栈 */
font-family: 'JetBrains Mono', 'Fira Code',
  'SF Mono', 'Cascadia Code', monospace;
```

### 3.3 排版参数建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 正文字号 | 14px~16px | 知识类产品推荐 15px 或 16px |
| 行高 | 1.6~1.75 | 长文阅读场景需要更大行高 |
| 段落间距 | 1em~1.5em | 段落分明，减少视觉压力 |
| 最大内容宽度 | 680px~720px | 参考 Notion/Medium 的阅读宽度 |
| 标题层级比例 | 1.25 比例 | H1: 2em → H2: 1.6em → H3: 1.28em |

## 4. 视觉风格趋势

### 4.1 布局趋势

| 趋势 | 描述 | 参考产品 |
|------|------|---------|
| **Bento Grid** | 圆角矩形卡片网格布局，模块化信息展示 | Apple、Linear |
| **侧边栏 + 主内容区** | 左侧导航/文件树 + 右侧内容阅读区的经典布局 | Notion、Obsidian |
| **命令面板（Command Palette）** | `Cmd+K` 全局搜索/命令入口，替代复杂导航 | Notion、Linear、Raycast |
| **面板化（Panel-based）** | 多面板并排查看，适合知识对比和溯源 | Heptabase、Arc Browser |

### 4.2 风格趋势

| 风格 | 说明 |
|------|------|
| **极简功能主义** | 去除装饰性元素，每个像素都有功能价值。2026 IxDF 研究显示极简设计减少 20% 认知负荷 |
| **内容优先** | 界面为内容服务，空状态和引导清晰但不喧宾夺主 |
| **自适应界面** | 根据用户偏好、环境光线、使用时间自动调整配色和密度 |
| **克制的品牌表达** | 品牌色仅在关键交互点出现，不铺满界面 |

### 4.3 动效趋势

| 类型 | 用法 | 参数参考 |
|------|------|---------|
| **页面过渡** | 淡入淡出，避免滑动式动画（感觉沉重） | duration: 150~250ms, ease-out |
| **微交互** | 按钮按下缩放、Toggle 切换、加载骨架屏 | duration: 100~200ms |
| **列表/卡片进入** | 渐次出现（stagger），从上到下依次淡入 | stagger: 30~50ms, fade + translateY(8px) |
| **AI 回答流式输出** | 打字机效果 + 光标闪烁，传达「AI 正在思考」 | 逐字/逐句渲染 |
| **溯源跳转** | 高亮闪烁原文段落，引导用户视线 | 黄色高亮渐变消失，duration: 1.5s |

### 4.4 圆角趋势

| 元素 | 推荐圆角 | 说明 |
|------|---------|------|
| 按钮 | 8px | 中等圆角，友好但不幼稚 |
| 卡片 | 12px | Bento Grid 风格 |
| 输入框 | 8px | 与按钮统一 |
| 弹窗/模态 | 16px | 更大圆角增强层级感 |
| 头像/图标容器 | 50%（圆形） | 用户头像 |
| 标签/Badge | 6px | 紧凑元素用小圆角 |

## 5. 风格参考案例（文字情绪板）

### 情绪板 A：「宁静书房」

> 想象一间北欧风格的书房。浅木色书桌上放着一杯咖啡，窗外是柔和的自然光。墙壁是温暖的米白色，书架上整齐排列着各色书脊。空间安静但不冷清，一切都在它应该在的位置。

**关键词**：温暖、有序、阅读感、自然光线、呼吸感
**对应方案**：方案二「温暖知识」
**参考产品**：Readwise Reader、Craft

### 情绪板 B：「未来实验室」

> 一间深色调的工作室，大屏幕上流动着蓝紫色的数据可视化。界面干净利落，每个元素都精确到像素。AI 助手的回答像流水一样在屏幕上逐行显现，附带精确的来源引用。氛围是专业的、高效的、令人信赖的。

**关键词**：科技感、精确、信任、深色、数据驱动
**对应方案**：方案一「深邃智慧」
**参考产品**：Linear、Raycast、ChatGPT

### 情绪板 C：「无印白纸」

> 一张没有多余线条的白纸。文字是唯一的主角，黑色墨水印在纯白纸面上。偶尔出现的蓝色下划线是唯一的色彩，暗示着「这里有更多内容可以探索」。极致的克制，极致的专注。

**关键词**：极简、专注、黑白、内容即界面、零干扰
**对应方案**：方案三「极简专注」
**参考产品**：Notion、iA Writer、Bear

### 情绪板 D：「知识星图」

> 深色宇宙般的背景上，知识节点像星星一样散布。它们之间的连线像星座，当你点亮一颗星，相关的星星也随之亮起。每一次搜索就像在星空中导航，每一个回答都能追溯到某颗具体的星。

**关键词**：探索感、图谱化、深色、发光、连接
**适用场景**：知识图谱可视化页面、Onboarding 引导
**参考产品**：Obsidian Graph View、Recall.ai

### 情绪板 E：「咖啡与笔记」

> 街角咖啡馆的午后。暖色灯光下，你在翻看平板上的笔记。界面像一本精装笔记本，米色纸张上是清晰的字迹。偶尔的绿色标记提醒你：「这段内容和你昨天看的那篇文章有关联」。

**关键词**：亲切、日常、轻松、笔记本质感、低压力
**适用场景**：移动端体验、轻度使用场景
**参考产品**：Apple Notes、Day One

## 6. 设计红线

### 6.1 色彩红线

| 红线 | 原因 |
|------|------|
| 主色不得同时超过 2 种 | 知识类产品需要克制，多色杂乱分散注意力 |
| 文字与背景对比度必须 >= 4.5:1（WCAG AA） | 长时间阅读场景对可读性要求极高 |
| 暗色模式不能简单反转色值 | 需要单独设计暗色配色，背景不能用纯黑 `#000000`（推荐 `#0A0A0A` ~ `#111827`） |
| 错误色（红色）不能用作装饰色 | 红色在知识库产品中应严格保留给错误/删除等操作 |
| 高亮/溯源色不能与品牌主色相同 | 溯源高亮需要独立的视觉通道，推荐黄色系 `#FEF3C7` ~ `#FCD34D` |

### 6.2 字体红线

| 红线 | 原因 |
|------|------|
| 正文字号不得小于 14px | 知识阅读场景要求高可读性 |
| 行高不得低于 1.5 | 密集文字阅读需要足够呼吸空间 |
| 不使用衬线体作为 UI 字体 | 屏幕渲染效果差，与现代 UI 风格不符（内容正文可例外） |
| 中英文必须分别指定字体 | 避免中文 fallback 到系统默认字体导致不一致 |
| 字重使用不超过 3~4 级 | Regular(400)、Medium(500)、SemiBold(600)、Bold(700) 足矣 |

### 6.3 布局红线

| 红线 | 原因 |
|------|------|
| 阅读区宽度不超过 720px | 过宽影响阅读节奏（参考 65~75 字符/行最佳阅读体验） |
| 移动端不使用侧边栏常驻 | 屏幕空间宝贵，使用抽屉式导航 |
| AI 对话区域与文档区域需要视觉区隔 | 用户需要明确分辨「AI 说的」和「我的资料」 |
| 溯源引用必须可点击跳转 | 仅显示来源但不可交互会破坏信任 |
| 加载状态必须有视觉反馈 | AI 回答可能需要 3~10 秒，无反馈用户会认为系统卡死 |

### 6.4 交互红线

| 红线 | 原因 |
|------|------|
| 文件上传必须有进度反馈 | 大文件上传无反馈是极差体验 |
| 不做无法撤销的操作而没有确认 | 删除知识条目需要二次确认 |
| 动效时长不超过 300ms | 效率工具用户对响应速度敏感 |
| 不在正文区域使用弹窗广告或升级提示 | 打断阅读/问答流程会导致用户流失 |
