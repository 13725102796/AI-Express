# 研发人效 Dashboard 设计色彩报告

> 调研时间：2026-04-09
> 目标用户画像：CTO/工程总监/组长/杰森集团管理层，30-50岁，偏好高效、专业、不花哨的工具界面
> 产品调性关键词：专业可信、清晰高效、数据驱动、管理层友好

## 1. 行业色彩语言

### 1.1 行业色彩惯例

研发效能/数据仪表盘领域有明确的色彩传统：

- **深蓝/靛蓝主色调**：传达专业、可信赖、技术深度。LinearB、Jellyfish、Swarmia 均以蓝色系为主色
- **暗色背景趋势**：2026 年 Dashboard 行业已将暗色模式视为标配，暗色背景能让数据图表更突出。Grafana 以暗色为默认 — 来源：[Muzli Dashboard Design 2026](https://muz.li/blog/best-dashboard-design-examples-inspirations-for-2026/)
- **高饱和度图表色**：数据可视化需要 5-7 个高区分度的色彩用于图表系列 — 来源：[Julius AI - Colors for Data Visualization](https://julius.ai/articles/best-colors-for-data-visualization)
- **语义色标准化**：成功=绿、警告=黄/橙、错误=红 是全球通用的语义色约定
- **亮色侧边栏 + 暗色内容区**或反之，用色彩反差建立导航层级 — 来源：[AdminLTE Dashboard Color Schemes](https://adminlte.io/blog/best-admin-dashboard-color-schemes/)

### 1.2 头部产品用色统计

| 产品 | 主色 | 辅色 | 强调色 | 整体风格 |
|------|------|------|--------|---------|
| LinearB | #3B82F6（亮蓝） | #1E293B（暗灰蓝） | #10B981（翠绿） | 专业蓝+暗底，技术感 |
| Jellyfish | #6366F1（靛紫蓝） | #F8FAFC（极淡灰） | #F59E0B（琥珀） | 商务感，亮色为主 |
| Swarmia | #2563EB（宝蓝） | #F1F5F9（暖灰白） | #EF4444（红色强调） | 简洁明快，数据友好 |
| Grafana | #FF6600（Grafana 橙） | #1A1B1F（深灰黑） | #73BF69（图表绿） | 暗色技术风，橙色品牌色醒目 |
| Plane | #3F76FF（Plane 蓝） | #F5F5F5（浅灰） | #FFBA18（金黄） | 现代 SaaS 风，明亮 |

**行业色彩总结**：蓝色系占绝对主导地位（4/5 产品以蓝为主色），暗灰/深灰蓝为通用辅色，强调色多用绿或橙/黄。

### 1.3 色彩心理学参考

目标用户群体（管理层/CTO/组长，30-50岁）的色彩偏好：
- **偏好**：深蓝（可信赖）、深灰（专业稳重）、白色（简洁高效）
- **可接受**：靛蓝/深紫蓝（现代感）、深绿（成功/增长）
- **文化禁忌**：纯红色大面积使用（中国语境下有"亏损/危险"暗示，仅限语义色用于错误/预警）
- **注意**：管理层对"技术极客"审美（暗色霓虹、代码编辑器风格）可能不感冒，偏好商务/报告质感

## 2. 推荐配色方案（3 套）

### 方案 A："信赖靛蓝"（Trusted Indigo）

- **调性关键词**：专业、可信赖、管理层友好
- **适合场景**：面向非技术管理层的数据展示，商务报告质感
- **色彩体系**：
  - 主色：#3B5998 — 用途：品牌色/主导航/主按钮，稳重的靛蓝（非纯蓝，带一丝暖调）
  - 辅色：#64748B — 用途：次要操作/分区背景/辅助文字
  - 强调色：#F59E0B — 用途：CTA/高亮/关键数据强调，琥珀色从蓝色背景中跳出
  - 成功色：#059669 — 用途：完成状态/正向指标
  - 警告色：#D97706 — 用途：预警/即将到期
  - 错误色：#DC2626 — 用途：失败/逾期/阻塞
  - 背景色（亮）：#F8FAFC — 微暖白，不刺眼
  - 背景色（暗）：#0F172A — 深海蓝黑，非纯黑
  - 文字主色：#1E293B — 深灰蓝，不是纯黑
  - 文字次色：#64748B — 中灰蓝，带色调
- **对比度检查**：主色 #3B5998 在 #F8FAFC 白底上对比度 5.8:1（AA 通过），在 #0F172A 暗底上对比度 3.2:1（大文字 AA 通过）
- **参考来源**：LinearB + Jellyfish 色系融合，偏商务质感
- **使用建议**：60% 白/暗底 + 30% 靛蓝导航和卡片头 + 10% 琥珀色关键数据

### 方案 B："Moonlit 深空"（Moonlit Space）

- **调性关键词**：现代、沉浸、数据可视化优先
- **适合场景**：以暗色为主的数据密集型仪表盘，图表优先
- **色彩体系**：
  - 主色：#6366F1 — 用途：品牌色/主按钮/图表主系列，靛紫蓝（现代感强）
  - 辅色：#818CF8 — 用途：次要操作/hover 态/图表辅系列
  - 强调色：#34D399 — 用途：CTA/成功态/正向数据，翠绿从暗底跳出
  - 成功色：#10B981 — 用途：完成状态
  - 警告色：#FBBF24 — 用途：预警
  - 错误色：#F87171 — 用途：失败/错误（暗底上用浅红）
  - 背景色（亮）：#F1F5F9 — 极淡蓝灰
  - 背景色（暗）：#1E1B2E — 深紫灰，带品牌色调
  - 文字主色：#E2E8F0 — 暗色模式用浅灰白
  - 文字次色：#94A3B8 — 中等灰蓝
- **对比度检查**：主色 #6366F1 在 #1E1B2E 暗底上对比度 4.7:1（AA 通过），强调色 #34D399 在暗底上对比度 8.2:1（AAA 通过）
- **参考来源**：Grafana 暗色风格 + 现代 SaaS 配色
- **使用建议**：60% 深色背景 + 30% 靛紫系卡片/组件 + 10% 翠绿 CTA

### 方案 C："沉稳墨石"（Ink Stone）

- **调性关键词**：沉稳、高端、报告感、东方商务美学
- **适合场景**：偏向企业内部报告风格，管理层审阅文档的质感
- **色彩体系**：
  - 主色：#334155 — 用途：品牌色/导航/主按钮，墨石蓝灰（极度沉稳）
  - 辅色：#475569 — 用途：次要区域/分区
  - 强调色：#0EA5E9 — 用途：CTA/数据高亮/关键指标，天蓝从灰调中跳出
  - 成功色：#059669 — 用途：完成状态
  - 警告色：#D97706 — 用途：预警
  - 错误色：#DC2626 — 用途：失败
  - 背景色（亮）：#FAFAF9 — 微暖白（偏暖，纸张感）
  - 背景色（暗）：#18181B — 墨色，非纯黑
  - 文字主色：#1C1917 — 深墨色
  - 文字次色：#78716C — 暖灰
- **对比度检查**：主色 #334155 在 #FAFAF9 上对比度 8.5:1（AAA 通过），强调色 #0EA5E9 在白底上对比度 3.1:1（大文字 AA，需配合深色文字）
- **参考来源**：Apple 报告风格 + 中式商务审美融合
- **使用建议**：60% 暖白底 + 30% 墨石灰导航和卡片 + 10% 天蓝高亮

### 方案对比与推荐

| 维度 | 方案 A 信赖靛蓝 | 方案 B Moonlit 深空 | 方案 C 沉稳墨石 |
|------|----------------|-------------------|----------------|
| 调性 | 专业商务 | 现代沉浸 | 高端报告 |
| 行业契合度 | 5/5 | 4/5 | 3/5 |
| 差异化程度 | 2/5（太安全） | 4/5（暗色系差异化） | 3/5 |
| 无障碍友好度 | 5/5 | 4/5 | 4/5 |
| 管理层接受度 | 5/5 | 3/5（管理层可能嫌暗） | 4/5 |
| 图表可视化效果 | 4/5 | 5/5 | 3/5 |

**首选推荐**：方案 A（信赖靛蓝），理由：
1. 目标用户是管理层/CTO，专业商务调性接受度最高
2. 亮色为主的方案在数据图表和文字可读性上更安全
3. 行业契合度最高，用户不需要"学习"界面风格
4. 可在此基础上增加暗色模式作为可选项

**如果追求差异化**：方案 B 是更有设计感的选择，但需要确认管理层对暗色界面的接受度。

## 3. 字体趋势

### 3.1 行业字体风格

研发效能 Dashboard 行业字体特征：
- **无衬线占绝对主流**：所有竞品均使用无衬线字体，传达现代、清晰、技术感
- **几何型无衬线偏好**：LinearB/Swarmia 使用几何感强的字体，字形规整
- **数据展示需要等宽数字**：KPI 数字需要 tabular figures（等宽数字）避免数值跳动
- **中文字体需求**：需要高质量中文无衬线字体配合

### 3.2 推荐字体搭配

| 用途 | 推荐字体 | 备选 | 说明 |
|------|---------|------|------|
| 英文标题 | Plus Jakarta Sans | Outfit, Onest | 几何感强、有辨识度，不是 Inter/Roboto 烂大街 |
| 英文正文 | Plus Jakarta Sans（400 weight） | Figtree | 同一字体多字重即可，减少加载体积 |
| 中文标题 | HarmonyOS Sans SC | 思源黑体 Noto Sans SC | 华为开源字体，现代感强，免费可商用 |
| 中文正文 | HarmonyOS Sans SC（Regular） | 思源黑体 Regular | 阅读舒适度好 |
| 代码/等宽 | JetBrains Mono | Fira Code | 编程字体，Git SHA/代码片段展示用 |
| 数字/KPI | Plus Jakarta Sans（tabular figures） | — | 确保开启 font-feature-settings: "tnum" 等宽数字 |

## 4. 视觉风格趋势

### 4.1 行业视觉趋势

**布局模式**：
- **侧边栏导航（占 80%+）**：Dashboard 类产品几乎全部使用左侧固定侧边栏 + 右侧内容区
- **顶部筛选栏 + 网格面板**：总览页通常为 2-3 列网格排列图表面板
- **卡片化信息组织**：每个指标/图表用卡片包裹，卡片之间等间距排列

**视觉风格**：
- **2026 主流：轻量级扁平 + 微妙阴影**：不是 2015 的纯扁平，也不是 glassmorphism 过度渲染，而是柔和多层阴影提供层级感 — 来源：[Muzli 2026](https://muz.li/blog/best-dashboard-design-examples-inspirations-for-2026/)
- **数据密度适中**：每个面板专注一个指标，不堆砌。留白充足
- **颜色节制**：Dashboard 背景以中性色为主，颜色仅用于数据区分和语义（成功/警告/错误）

**暗色/亮色**：
- **双模式成为标配**：73% 的用户期望暗色模式支持 — 来源：[Medium Dark Mode Best Practices](https://medium.com/@social_7132/dark-mode-done-right-best-practices-for-2026-c223a4b92417)
- **建议**：默认亮色（管理层习惯），提供暗色模式切换

**动效风格**：
- **克制微交互**：Dashboard 不适合花哨动画，数据加载淡入、图表过渡动画即可
- **图表入场动画**：ECharts 默认提供的渐入动画足够
- **hover 反馈**：卡片 hover 微微上浮（translateY -2px + 阴影加深）

**圆角趋势**：
- **中等圆角（8-12px）**：不是极大圆角（那是 C 端），也不是直角（太生硬）
- 按钮 8px、卡片 12px、输入框 8px 是 2026 年 Dashboard 的主流

### 4.2 风格方向建议

- **推荐风格**：轻量级扁平 + 柔和阴影 + 数据优先 — 适合管理层快速扫描数据，不被界面元素干扰
- **暗色/亮色**：默认亮色模式，可选暗色 — 依据：管理层用户更习惯亮色界面
- **圆角策略**：按钮 8px、卡片 12px、弹窗 16px — 依据：Dashboard 行业主流
- **图表风格**：ECharts 默认主题微调，柱状图/折线图/环形图为主，避免 3D 图表

## 5. 风格参考案例（文字情绪板）

### 案例 1：LinearB Dashboard

- **整体感受**：专业、数据密集、工程师友好、有深度
- **配色特点**：暗蓝灰底 + 白色卡片，蓝绿为图表主色，信息密度高
- **排版特点**：紧凑行距、小字号（13-14px 正文）、多层级标题
- **组件风格**：圆角小（6-8px）、轻薄卡片、无阴影或极淡阴影
- **动效特点**：极简，几乎无动效，数据快速加载
- **值得借鉴**：指标卡片的信息密度——一个卡片同时展示当前值、趋势箭头和同比
- **需要规避**：字号过小、信息过密（管理层扫描效率低）

### 案例 2：Swarmia Dashboard

- **整体感受**：清爽、现代、一目了然
- **配色特点**：白底为主、宝蓝主色、图表用绿红蓝三色、留白充足
- **排版特点**：字号适中（15-16px 正文）、层级清晰、间距舒适
- **组件风格**：中圆角（8-12px）、卡片有微妙阴影、按钮有呼吸感
- **动效特点**：图表平滑过渡、页面切换有淡入效果
- **值得借鉴**：留白和信息密度的平衡——每个面板只聚焦一个指标，管理层扫一眼就懂
- **需要规避**：强调色使用过少（CTA 不够突出）

### 案例 3：Grafana Dashboard

- **整体感受**：技术感强、暗色沉浸、高度可定制
- **配色特点**：深灰黑底、橙色品牌色、图表用高饱和多色系列
- **排版特点**：等宽字体混用、字号偏小、面板标题紧凑
- **组件风格**：直角或小圆角、面板边框明确、工具感强
- **动效特点**：实时数据刷新动画、图表 hover tooltip 响应快
- **值得借鉴**：暗色背景让图表颜色非常突出，数据可视化效果顶级
- **需要规避**：技术极客审美不适合管理层用户，过于"工程师工具"

### 案例 4：Plane 项目管理界面

- **整体感受**：现代 SaaS、亲和力强、操作直觉
- **配色特点**：白底 + Plane 蓝主色 + 金黄强调色，色彩运用克制
- **排版特点**：适中字号、行间距舒适、Sidebar + 内容区经典布局
- **组件风格**：中圆角、柔和阴影、按钮有明确层级（Primary/Secondary/Ghost）
- **动效特点**：拖拽交互流畅、弹窗渐入渐出、列表项微动画
- **值得借鉴**：与 Plane 同一设计语言家族可以降低用户学习成本
- **需要规避**：Plane 是操作工具，Dashboard 是只读展示，交互模式应不同

## 6. 设计红线

### 必须避免

- **将 Dashboard 设计成"监控大屏"风格**：深蓝/黑底+大荧幕+闪烁数字 = 工厂监控室，不是管理层工具
- **数据密度过高**：每个面板不超过一个核心指标+一个图表，不要在一个卡片里堆砌多个数据
- **过度使用红色**：红色仅限错误/逾期/阻塞语义，不要用红色做强调色或品牌色
- **ECharts 默认配色直接使用**：ECharts 默认配色太"模板感"，需要用品牌色系重新定义图表调色板

### 色彩禁忌

- **纯黑 #000000**：不用纯黑，深色用 #0F172A 或 #18181B（带色调）
- **纯白 #FFFFFF**：背景用 #F8FAFC 或 #FAFAF9（微暖白）
- **纯灰色**：所有灰度必须带品牌色调（偏蓝灰或偏暖灰）
- **高饱和度大面积使用**：强调色只在 10% 面积出现，大面积使用降低饱和度
- **红绿混用无辅助标识**：考虑色盲用户，红绿除颜色外需配合图标/文字

### 风格陷阱

- **Glassmorphism 毛玻璃效果**：在 Dashboard 里增加视觉噪音，降低数据可读性
- **渐变背景**：管理层工具不需要"氛围感"，纯色背景更专业
- **卡片嵌套**：卡片里套卡片是视觉噪音，一层卡片足够
- **动画过度**：Dashboard 不需要入场大动画，快速呈现数据是第一优先级
- **"科技感"暗色霓虹**：目标用户不是开发者，不需要"技术酷炫"，需要"专业可信"
