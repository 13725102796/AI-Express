# DevPerf Dashboard 操作文档

> 从零开始的完整操作流程，按顺序执行即可。

---

## 前置条件

- 后端已启动：http://localhost:3200
- 前端已启动：http://localhost:5173
- Gitea 服务已部署并可访问（如果要同步 Git 数据）

---

1. 浏览器打开 http://localhost:5173
2. 输入管理员账号：
   - 邮箱：`admin@jasonqiyuan.com`
   - 密码：`Admin123!`
3. 点击「Sign In」
4. 登录成功后进入团队总览页（此时是空的，因为没有数据）

## 第二步：创建开发人员

1. 点击左侧导航栏「Admin」
2. 在 Users 标签页下，点击「Create User」按钮
3. 逐个创建你的团队成员：

**示例 - 创建第一个开发人员：**

| Role | developer |
| Git Username | chenqiang（填 Gitea 上的用户名） |

4. 点击确认创建
5. 重复以上步骤，创建所有团队成员

## 第三步：配置 Git 作者映射

> 这一步确保 Gitea 上的 commit/PR 能正确关联到 Dashboard 中的用户。

1. 还是在「Admin」页面，点击「Author Mapping」标签
2. 系统在同步 Git 数据后，会自动尝试匹配（按邮箱）
3. **如果自动匹配失败**（比如 Git 邮箱和系统邮箱不一样），你需要手动添加映射：
   - 点击「Add Mapping」
   - Git Email：填开发人员在 Git 上配置的邮箱（`git config user.email` 看到的）
   - Git Username：填 Gitea 用户名
   - Mapped User：选择对应的系统用户

> **举例**：如果陈强在 Git 上的邮箱是 `cq@gmail.com`，但 Dashboard 里注册的是 `chenqiang@jasonqiyuan.com`，你就需要添加一条映射：
> `cq@gmail.com` → 陈强

## 第四步：配置 Gitea 连接并同步

### 4.1 配置环境变量

编辑后端的 `.env` 文件（`code/backend/.env`）：

```bash
# 改成你的 Gitea 实际地址
GITEA_BASE_URL=http://你的Gitea地址:3000

# 在 Gitea 上生成 API Token：
# 登录 Gitea → 右上角头像 → Settings → Applications → Generate New Token
GITEA_API_TOKEN=你的token

# 你的 Gitea 组织名
GITEA_ORG=jasonqiyuan

# 同步频率（分钟）
SYNC_GITEA_INTERVAL=30
```

### 4.2 重启后端

改完 `.env` 后需要重启后端服务，让新配置生效。

### 4.3 手动触发同步

1. 回到 Dashboard 的「Admin」页面
2. 点击「Sync Logs」标签
3. 点击「Trigger Sync」按钮
4. 等待几秒后刷新页面，查看同步日志：
   - `gitea / success` = 同步成功
   - `gitea / error` = 同步失败（检查 .env 配置和网络连接）

### 4.4 验证同步结果

同步成功后：
- 点击左侧「Git Activity」→ 应该能看到团队的提交记录和 PR 数据
- 贡献热力图应该有色块
- 统计卡片应该有数字（Total PRs、Merged、Avg Merge Time）

> **后续自动同步**：配置好后，系统每 30 分钟自动从 Gitea 拉取新数据，不需要手动操作。

## 第五步：配置 Plane 连接并同步

> 如果你用 Plane 管理任务，这一步让 Dashboard 自动拉取项目和任务数据。
> 如果不用 Plane，跳过此步。

### 5.1 配置环境变量

编辑 `code/backend/.env`：

```bash
PLANE_BASE_URL=http://你的Plane地址:8082
PLANE_API_TOKEN=你的PlaneAPIToken
PLANE_WORKSPACE_SLUG=jasonqiyuan
SYNC_PLANE_INTERVAL=15
```

> Plane API Token 生成方式：登录 Plane → Settings → API Tokens → Create API Token

### 5.2 触发同步

同样在 Admin → Sync Logs → Trigger Sync

### 5.3 同步后的效果

- 「Overview」团队总览页会显示：
  - Sprint 交付率趋势（从 Plane 的 Cycle 数据算出）
  - 任务状态分布（todo/in_progress/review/done）
  - 各产品线当前 Sprint 进度
- 「Projects」会列出所有 Plane 项目
- 点进某个项目能看到燃尽图、里程碑、任务矩阵

## 第六步：创建 OKR

> OKR 是在 Dashboard 中直接管理的，不来自 Plane。

1. 点击左侧「OKR」
2. 需要用 **manager 或 admin** 账号操作

### 6.1 创建 Objective（目标）

点击页面上的创建按钮（如果前端没有创建按钮，可以用 API）：

```bash
# 用 curl 创建（替换 TOKEN 为登录后拿到的 JWT）
curl -X POST http://localhost:3200/api/okr/objectives \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你的TOKEN" \
  -d '{
    "title": "Q2 完成 Avatar 2.0 核心功能开发",
    "ownerId": "对应用户的ID",
    "projectId": "对应项目的ID",
    "period": "2026-Q2"
  }'
```

> **获取 TOKEN**：打开浏览器开发者工具 → Network → 随便点一个页面请求 → 看 Request Headers 里的 `Authorization: Bearer xxx`，复制 xxx 部分。

> **获取用户 ID 和项目 ID**：Admin 页面的用户列表、Projects 页面都能看到，或者用 API 查询。

### 6.2 创建 Key Result（关键结果）

```bash
curl -X POST http://localhost:3200/api/okr/objectives/目标ID/key-results \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你的TOKEN" \
  -d '{
    "title": "Sprint 交付率达到 85%",
    "targetValue": 85,
    "unit": "%",
    "weight": 2
  }'
```

### 6.3 更新 KR 进度

```bash
curl -X PATCH http://localhost:3200/api/okr/key-results/KR的ID \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你的TOKEN" \
  -d '{ "currentValue": 72 }'
```

更新后 Objective 的总进度会自动重算（加权平均）。

## 常见问题

### Q: Git 同步后看不到数据？
1. 检查 Admin → Sync Logs 是否有 error 记录
2. 检查 `.env` 中的 `GITEA_BASE_URL`、`GITEA_API_TOKEN`、`GITEA_ORG` 是否正确
3. 检查 Author Mapping 是否正确（Git 用户名 → 系统用户）

### Q: Plane 同步后项目进度是 0%？
项目进度只看 **当前 active Sprint** 的完成率。如果 Plane 上没有设置 active 的 Cycle，进度就是 0%。

### Q: OKR 进度怎么更新？
目前需要 manager/admin 手动更新 KR 的 currentValue。更新后 Objective 进度自动重算。

### Q: 怎么给杰森集团创建只读账号？
Admin → Create User → Role 选「viewer」→ 把账号密码给对方。viewer 只能看总览和项目，不能改任何东西。
