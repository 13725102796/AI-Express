# 天机 -- AI 算命 产品需求文档（PRD）

## 1. 产品概述

- **产品名称**：天机 -- AI 算命
- **一句话定位**：一个有古风沉浸感的 AI 对话式算命小工具，输入生辰八字即可与"算命大师"聊天，获取运势、性格、事业、婚姻分析，纯娱乐、不贩卖焦虑。
- **目标用户**：20-30 岁年轻人，对中国传统文化有亲近感，把算命当作轻松娱乐和社交话题
- **核心价值**：用最快的路径（输入生日 -> 对话算命 -> 分享结果）获得一次有趣的"算命体验"，视觉沉浸感让整个过程像真的去找了一位仙风道骨的大师问卦

## 2. 用户画像

| 维度 | 描述 |
|------|------|
| 角色 | 20-30 岁年轻人（以女性为主，占比约 60-70%），大学生或初入职场 |
| 痛点 | 1. 现有算命 App 广告多、界面老旧、付费套路深 2. AI 算命结果模板化，千人一面，不好玩 3. 遇到人生困惑时想找个"心理出口"但不想花大钱 |
| 期望 | 1. 打开就能用，不需要注册和填长表单 2. 有趣的对话体验，能追问细节 3. 古风氛围感，像真的在问大师 4. 结果正面积极，不贩卖焦虑 |
| 使用场景 | 1. 无聊/焦虑时打开算一卦解闷 2. 朋友聚会时互相算命、对比结果 3. 新年/生日等节点看看来年运势 4. 面临选择（求职/恋爱）时找点"玄学参考" |

## 3. 功能需求

### 3.1 生辰八字输入（P0）

- **描述**：用户输入出生年、月、日、时，系统自动计算天干地支八字
- **用户故事**：作为一个想算命的用户，我希望快速输入我的出生时间，以便开始算命体验
- **验收标准**：
  - Given 用户打开应用进入主页
  - When 用户通过滚轮/选择器输入出生的年、月、日、时辰
  - Then 系统在 500ms 内显示对应的天干地支八字，并自动跳转到算命对话页
- **优先级**：P0
- **边界条件与异常场景**：
  - 时辰可选"不确定"，此时系统根据日期进行简化分析并提示"如提供时辰可获得更准确的结果"
  - 日期范围限制在 1940-2026 年
  - 不强制要求姓名、性别等信息（减少隐私顾虑），但可选填性别以优化结果

### 3.2 AI 对话式算命（P0）

- **描述**：基于用户八字，AI 以"算命大师"角色进行对话式算命分析，用户可自由追问
- **用户故事**：作为一个已输入生辰八字的用户，我希望与 AI 算命大师对话交流，以便获得个性化的运势分析并追问感兴趣的细节
- **验收标准**：
  - Given 用户已输入生辰八字
  - When 系统加载完成
  - Then AI 主动发出第一条消息（包含八字排盘结果和整体运势概述），响应时间 < 3 秒
  - Given AI 已发出第一条消息
  - When 用户输入追问（如"我的事业运怎么样"）
  - Then AI 在 3 秒内回复针对性分析，措辞风格保持"古风大师"角色一致
- **优先级**：P0
- **边界条件与异常场景**：
  - AI 回复失败时显示"大师正在卜算中..."重试提示，最多重试 2 次
  - 单次对话上限 20 轮（超出提示"今日卦象已尽，明日再来"）
  - AI 回复必须包含"仅供娱乐参考"的免责标识（底部灰色小字）
  - AI 不得给出明确的负面断言（如"你会离婚"），而是用正向引导语（如"感情方面宜多沟通、珍惜缘分"）

### 3.3 算命结果展示（P0）

- **描述**：AI 首次回复时，以结构化方式展示八字分析结果，涵盖多个人生维度
- **用户故事**：作为一个想了解自己运势的用户，我希望看到清晰的多维度分析结果，以便对自己的各方面运势有全面了解
- **验收标准**：
  - Given AI 完成首轮分析
  - When 系统渲染结果
  - Then 结果以卡片形式分维度展示，包含：总体运势（1 段）、性格分析（1 段）、事业运（1 段）、感情运（1 段）、财运（1 段），每段 50-100 字
  - Given 结果已展示
  - When 用户点击某个维度卡片
  - Then 展开该维度的详细分析（150-300 字）
- **优先级**：P0
- **边界条件与异常场景**：
  - 结果始终保持正面积极的基调，不出现恐吓性内容
  - 每个维度配有对应的古风小图标（如事业用"印章"、感情用"鸳鸯"）

### 3.4 结果分享（P1）

- **描述**：用户可将算命结果生成古风风格的分享卡片/签文图片，分享到社交平台
- **用户故事**：作为一个得到有趣算命结果的用户，我希望生成一张好看的古风签文图片，以便分享到朋友圈引发互动
- **验收标准**：
  - Given 用户已完成算命对话
  - When 用户点击"生成签文"按钮
  - Then 系统在 2 秒内生成一张 9:16 比例的古风签文图片，包含：运势关键词、一句签诗、产品 logo 水印
  - Given 签文图片已生成
  - When 用户点击"分享"按钮
  - Then 调起系统分享面板（微信/朋友圈/保存图片）
- **优先级**：P1
- **边界条件与异常场景**：
  - 签文内容始终正面积极（不生成负面签文）
  - 图片生成失败时提供重试按钮
  - 分享图片不包含用户生辰八字等隐私信息

### 3.5 历史记录（P2）

- **描述**：本地存储用户的算命历史记录，方便回顾
- **用户故事**：作为一个多次使用的用户，我希望能回顾之前的算命结果，以便对比不同时间的运势变化
- **验收标准**：
  - Given 用户已完成至少 1 次算命
  - When 用户打开"历史签文"页面
  - Then 按时间倒序展示历史记录列表，每条显示日期和运势关键词
- **优先级**：P2
- **边界条件与异常场景**：
  - 数据存储在本地（localStorage），清除浏览器/App 数据后丢失
  - 最多保存 50 条记录，超出后覆盖最早的记录

## 4. 非功能需求

- **性能要求**：
  - 首屏加载 < 2 秒（4G 网络）
  - AI 首次响应 < 3 秒
  - 签文图片生成 < 2 秒
- **安全要求**：
  - 不收集用户姓名、手机号等身份信息
  - 生辰八字数据仅在本次会话中使用，不持久化到服务器
  - AI 对话内容不上传/存储（隐私优先）
  - 所有 API 通信使用 HTTPS
- **兼容性要求**：
  - 移动端优先（iOS Safari 15+、Android Chrome 90+）
  - 桌面端兼容（Chrome、Safari、Firefox 最新 2 个版本）
  - 最小屏幕宽度 320px
- **可访问性要求**：
  - 文字对比度 WCAG AA（4.5:1）
  - 支持 prefers-reduced-motion
  - 所有可交互元素最小点击区域 44x44px

## 5. 信息架构

```
天机 AI 算命
├── 首页（欢迎页 + 生辰输入）
│   ├── 品牌 Logo + 标语
│   ├── 生辰八字输入区（年/月/日/时辰选择器）
│   └── "开始算命" CTA 按钮
├── 算命对话页
│   ├── 八字排盘展示（顶部固定）
│   ├── AI 对话区域（滚动）
│   │   ├── AI 大师消息（古风气泡）
│   │   └── 用户消息
│   ├── 用户输入框（底部固定）
│   └── 操作栏：生成签文 / 重新算命
└── 签文分享页（叠加层）
```

**核心用户动线**：
1. 打开应用 -> 输入生辰 -> 点击"开始算命"
2. 进入对话页 -> 阅读 AI 首轮分析 -> 追问细节
3. 满意后 -> 点击"生成签文" -> 分享到社交平台

## 6. 设计指引

- **风格方向**：新中式暗色沉浸风（New Chinese Dark Immersive）
- **品牌调性关键词**：古风、神秘、高级感、沉浸、趣味、正能量
- **配色参考**：暗色底（墨褐黑/玄墨色）+ 金色强调色（赤金），参考"墨金雅韵"方案。金色仅在关键元素使用（CTA 按钮、重要高亮），保持稀缺感
- **视觉元素**：
  - 水墨渐变效果（轻量使用，不要满屏）
  - 八卦/太极图形（作为品牌图标元素）
  - 云雾/烟雾微动效（背景点缀，不干扰内容）
  - 传统纹样（回纹/云纹，仅用于边框或分隔线，克制使用）
- **字体**：标题使用有古韵的楷体/文楷字体（如 LXGW WenKai），正文使用衬线体（如 Noto Serif SC）保证可读性
- **动效基调**：缓慢、沉稳、有仪式感。入场动画 500-800ms，使用 exponential easing（ease-out-quart），不使用 bounce/elastic
- **禁止项**：不用紫蓝渐变、不用霓虹色、不用 glassmorphism、不用 emoji 作 UI 图标、不用大面积纯黑或纯白

## 7. 范围边界

- **MVP 包含**：
  - 生辰八字输入 + 八字排盘计算
  - AI 对话式算命（含系统 prompt 注入）
  - 多维度结果展示（运势/性格/事业/感情/财运）
  - 结果分享（生成签文图片）
  - 古风沉浸式 UI
  - 免责声明（"仅供娱乐参考"）

- **MVP 不包含（后续迭代）**：
  - 用户账号体系和登录
  - 历史记录（P2）
  - 每日运势推送
  - 情侣/朋友八字匹配
  - 付费功能（详细年度报告）
  - 多语言支持

- **明确排除**：
  - 真人大师咨询功能
  - 电商/广告模块
  - 社区/论坛功能
  - 面相/手相识别
  - 风水/择日等非八字功能

## 8. 开放问题

- AI 大模型选型：使用哪个大模型 API？（建议 DeepSeek 或 GPT-4o-mini，需评估成本和中文能力）
- 八字排盘算法：自研还是使用现有开源库？（建议调研 lunar-javascript 等开源方案）
- 产品形态：先做 Web App（H5）还是原生 App？（建议先做 H5，降低开发成本，验证 PMF 后再考虑原生）
- "娱乐免责声明"的法律合规性需进一步确认
