# AI-Express 产品需求文档（PRD）

## 1. 产品概述

- **产品名称**：AI-Express
- **一句话定位**：AI 驱动开发最佳实践知识库的 Web 展示平台，让开发者快速查阅、学习和应用 AI 辅助开发的 Skills、流程与经验
- **目标用户**：使用 AI 辅助编程的工程师（包括但不限于 Claude Code / Cursor / GitHub Copilot 用户）
- **核心价值**：将散落在 Markdown 文件中的 AI 驱动开发知识体系（Skills、经验总结、开发流程规范、项目角色 Agent 定义）转化为结构化、可检索、易浏览的 Web 界面，降低开发者系统化掌握 AI 辅助开发的门槛

## 2. 用户画像

| 维度 | 描述 |
|------|------|
| 角色 | 中高级软件工程师，已在日常开发中使用 AI 编程工具（Claude Code、Cursor、Copilot 等），希望从"能用"进阶到"用好" |
| 痛点 | 1. AI 编程工具的 Skills/Prompt 散落各处，没有统一的高质量收集和分类体系；2. 缺乏 AI 驱动开发的标准化流程，每次开发靠经验摸索；3. 同类经验总结以长文形式存在，难以快速定位所需知识点；4. 不清楚不同开发场景（新项目/增量/Bug修复/重构）应该使用哪些 Skills 和流程 |
| 期望 | 1. 有一个一站式平台，能按场景/分类快速找到合适的 Skills；2. 开发流程规范可视化，一眼看清从需求到交付的完整路径；3. 经验总结结构化呈现，支持搜索和标签过滤；4. 能快速复制 Skill 配置到自己的项目中使用 |
| 使用场景 | 1. 开始新项目前，查阅推荐的开发流程和 Skill 组合；2. 遇到特定开发场景时，搜索相关 Skills 和经验；3. 团队 onboarding 时，引导新成员学习 AI 驱动开发流程；4. 日常碎片时间浏览，发现新的优秀 Skills 和技巧 |

## 3. 功能需求

### 3.1 首页与全局导航

- **描述**：提供产品的全局入口和导航体系，包含 Hero 区域展示产品定位、核心模块快速入口、以及全站导航栏。首页作为知识库的门面，需要清晰传达产品价值并引导用户进入对应内容区域。
- **用户故事**：作为开发者，我希望打开网站首页就能快速了解 AI-Express 提供什么内容，并通过导航直达我感兴趣的模块，以便节省查找时间。
- **验收标准**：
  - Given 用户首次访问网站
  - When 页面加载完成
  - Then 展示 Hero 区域（产品名称、一句话定位、核心价值说明）、三大内容模块入口卡片（优秀 Skills、经验总结、开发流程）、顶部导航栏（包含各模块链接和全局搜索入口）
  - Given 用户点击顶部导航的某个模块链接
  - When 页面跳转
  - Then 平滑导航到对应模块页面，导航栏高亮当前所在模块
- **优先级**：P0
- **边界条件与异常场景**：
  - 移动端访问时导航栏折叠为汉堡菜单
  - 内容模块为空时，入口卡片显示"即将上线"状态
  - 页面加载超过 3 秒时展示骨架屏

### 3.2 Skills 展示与分类浏览

- **描述**：将收集的优秀 Skills 以卡片列表形式展示，支持按分类（开发场景、工具链、难度级别）筛选和排序。每个 Skill 卡片展示名称、一句话描述、适用场景标签、来源信息。点击卡片进入 Skill 详情页，展示完整的 Skill 定义、使用说明、配置代码和使用示例。
- **用户故事**：作为开发者，我希望能按开发场景（从 0 开发 / 增量功能 / Bug 修复 / 重构 / UI 开发）筛选出合适的 Skills，以便找到当前任务最匹配的工具。
- **验收标准**：
  - Given 用户进入 Skills 列表页
  - When 页面加载完成
  - Then 以卡片网格布局展示所有 Skills，每张卡片包含名称、描述摘要、场景标签，默认按推荐度排序
  - Given 用户点击场景筛选标签（如"Bug 修复"）
  - When 筛选生效
  - Then 列表仅显示标记为该场景的 Skills，筛选标签高亮，显示结果数量
  - Given 用户点击某个 Skill 卡片
  - When 进入详情页
  - Then 展示 Skill 完整信息：名称、描述、适用场景、完整配置代码（带语法高亮）、使用示例、相关 Skills 推荐
- **优先级**：P0
- **边界条件与异常场景**：
  - 筛选结果为空时，显示"未找到匹配的 Skills，试试其他筛选条件"并推荐热门 Skills
  - Skill 配置代码支持一键复制到剪贴板，复制成功后显示 Toast 提示
  - 长描述文本在卡片中截断显示，详情页中完整展示

### 3.3 开发流程可视化

- **描述**：将 `AI驱动开发流程规范.md` 中定义的多场景开发流程以可视化方式呈现。包括流程总览图、各场景（从 0 开发 / 增量开发 / 迭代优化 / Bug 修复 / 重构 / UI-UX / 长周期任务）的分步流程图、每个步骤的详细说明、以及流程选择决策树的交互式导航。
- **用户故事**：作为开发者，我希望通过可视化的流程图快速理解不同开发场景下应该遵循的步骤和使用的 Skills，以便在实际项目中按规范执行。
- **验收标准**：
  - Given 用户进入"开发流程"页面
  - When 页面加载完成
  - Then 展示流程总览图（Phase 0 到 Phase 4 的五阶段概览）和 7 个场景的入口卡片
  - Given 用户点击某个场景（如"增量功能开发"）
  - When 进入场景详情
  - Then 展示该场景的完整流程步骤图，每个步骤可展开查看详细说明、推荐的 Skills、质量门禁要求
  - Given 用户使用流程选择决策树
  - When 按提示回答问题（如"是新项目吗？""涉及 UI 变更吗？"）
  - Then 根据回答路径推荐最匹配的开发流程和 Skill 组合
- **优先级**：P0
- **边界条件与异常场景**：
  - 流程步骤中引用的 Skills 可点击跳转到对应 Skill 详情页
  - 移动端流程图支持横向滚动或切换为列表视图
  - 决策树回答过程中支持回退到上一步

### 3.4 经验总结浏览

- **描述**：将 `经验总结/` 目录下的知识内容以文章列表形式展示，支持按分类和标签过滤。每篇经验文章支持 Markdown 渲染，包含目录导航、代码块语法高亮。文章之间支持关联推荐。
- **用户故事**：作为开发者，我希望能浏览和搜索已沉淀的 AI 驱动开发经验，以便在遇到类似问题时快速找到解决方案和最佳实践。
- **验收标准**：
  - Given 用户进入"经验总结"页面
  - When 页面加载完成
  - Then 以列表形式展示所有经验文章，每篇显示标题、摘要、分类标签、发布日期
  - Given 用户点击某篇文章
  - When 进入文章详情页
  - Then 完整渲染 Markdown 内容，包含：页内目录导航（TOC）、代码块语法高亮、标题锚点链接、底部关联文章推荐
  - Given 用户点击分类标签进行筛选
  - When 筛选生效
  - Then 列表仅显示该分类下的文章
- **优先级**：P1
- **边界条件与异常场景**：
  - Markdown 中的外部链接在新窗口打开
  - 长文章在列表中仅显示前 150 字摘要
  - 文章内的代码块支持一键复制
  - 文章不存在或加载失败时显示 404 页面

### 3.5 全局搜索

- **描述**：提供全站内容搜索能力，覆盖 Skills、经验文章、流程文档等所有内容类型。支持关键词搜索，搜索结果按相关度排序，按内容类型分组展示，支持高亮匹配词。
- **用户故事**：作为开发者，我希望通过关键词搜索快速定位到与我当前问题相关的 Skills、流程或经验文章，以便在几秒内找到答案而非逐个模块翻阅。
- **验收标准**：
  - Given 用户点击导航栏的搜索图标或使用快捷键（Cmd/Ctrl + K）
  - When 搜索面板弹出
  - Then 展示搜索输入框和历史搜索记录（最近 5 条）
  - Given 用户输入关键词（如"代码审查"）并回车
  - When 搜索结果返回
  - Then 按内容类型分组展示结果（Skills / 流程 / 经验文章），每条结果包含标题、匹配片段（高亮关键词）、内容类型标签，点击可跳转到对应详情页
  - Given 搜索无结果
  - When 展示空状态
  - Then 显示"未找到相关内容"，并推荐热门内容
- **优先级**：P1
- **边界条件与异常场景**：
  - 搜索输入支持防抖（300ms），避免频繁请求
  - 支持搜索结果分页或无限滚动加载
  - 搜索关键词为空或仅空格时不触发搜索
  - 历史搜索记录存储在浏览器 localStorage 中

### 3.6 项目角色 Agent 展示

- **描述**：展示项目中定义的各类 Agent 角色（产品经理 Agent、设计 Agent、Phase 0 编排器等），说明每个 Agent 的职责、工作流程、输入输出物和使用方法。帮助用户理解 AI 驱动开发中的角色分工体系。
- **用户故事**：作为开发者，我希望了解 AI 驱动开发中有哪些 Agent 角色可以使用、每个角色做什么，以便在自己的项目中复用这套角色体系。
- **验收标准**：
  - Given 用户进入"角色 Agent"页面
  - When 页面加载完成
  - Then 以卡片形式展示所有 Agent 角色，每张卡片包含角色名称、一句话描述、所需工具列表、推荐模型
  - Given 用户点击某个 Agent 卡片
  - When 进入详情页
  - Then 展示完整的 Agent 定义：角色描述、核心原则、完整工作流程（分步骤）、输出物模板、使用示例，Agent 定义代码可一键复制
- **优先级**：P1
- **边界条件与异常场景**：
  - Agent 之间的协作关系以连线图或列表形式展示（如 phase0-orchestrator 调用 product-agent 和 design-agent）
  - Agent 定义文件中的 YAML frontmatter 解析展示为结构化信息

### 3.7 代码/配置一键复制

- **描述**：所有展示代码片段、Skill 配置、Agent 定义的位置均提供一键复制功能。复制内容为纯文本格式，保留原始缩进和格式。
- **用户故事**：作为开发者，我希望看到有用的 Skill 配置或代码片段时能一键复制到剪贴板，以便直接粘贴到我的项目中使用，而非手动选中复制。
- **验收标准**：
  - Given 用户看到代码块右上角的复制按钮
  - When 点击复制按钮
  - Then 代码内容复制到系统剪贴板，按钮图标短暂变为"已复制"状态（持续 2 秒后恢复）
  - Given 用户在不支持 Clipboard API 的浏览器中
  - When 点击复制按钮
  - Then 降级为选中代码文本并提示"请按 Cmd/Ctrl+C 复制"
- **优先级**：P0
- **边界条件与异常场景**：
  - 代码块支持语法高亮（YAML、Markdown、Bash、Python、JavaScript）
  - 复制功能通过 navigator.clipboard API 实现，降级方案使用 document.execCommand

## 4. 非功能需求

### 性能要求
- 首屏加载时间（FCP）不超过 1.5 秒（3G 网络下不超过 3 秒）
- 页面切换（SPA 路由跳转）不超过 300ms
- 搜索结果响应时间不超过 500ms
- Lighthouse Performance 评分不低于 90

### 安全要求
- 网站为纯静态站点，无用户认证和敏感数据处理
- Markdown 渲染需防范 XSS 攻击（对用户输入内容做 sanitize）
- 外部链接使用 `rel="noopener noreferrer"`
- 部署启用 HTTPS

### 兼容性要求
- 支持主流浏览器最近 2 个版本（Chrome、Firefox、Safari、Edge）
- 响应式设计，支持桌面端（1440px+）、平板（768px-1439px）、移动端（320px-767px）
- 支持系统级暗色模式（Dark Mode）

### 可访问性要求
- 语义化 HTML 标签（nav、main、article、section）
- 图片和图标提供 alt 文本
- 键盘导航支持（Tab 切换焦点、Enter 确认、Esc 关闭弹窗）
- 颜色对比度满足 WCAG 2.1 AA 标准

### SEO 要求
- 静态站点生成（SSG），确保搜索引擎可抓取
- 每个页面有独立的 title 和 meta description
- 支持 Open Graph 标签，便于社交媒体分享

### 核心用户动线

1. **场景驱动动线**：首页 → 开发流程 → 选择场景 → 查看推荐 Skills → 复制 Skill 配置
2. **搜索驱动动线**：任意页面 → Cmd+K 搜索 → 结果页 → 详情页 → 复制使用
3. **浏览驱动动线**：首页 → Skills 列表 → 分类筛选 → Skill 详情 → 相关推荐 → 继续浏览
4. **学习驱动动线**：首页 → 经验总结 → 文章详情 → 关联文章 → 对应流程/Skills

## 6. 设计指引

### 风格方向

面向开发者的技术知识库产品，视觉风格应体现**专业、高效、现代**的技术感。不追求花哨的视觉效果，而是通过清晰的信息层次、充足的留白和精准的排版传达"这是一个值得信赖的专业工具"。

整体风格倾向 **技术文档 + 现代 SaaS** 的交叉地带：比传统技术文档更有设计感，但比一般 SaaS 产品更克制、更聚焦内容。

### 参考产品

| 参考对象 | 借鉴要素 |
|---------|---------|
| **Vercel 文档站**（vercel.com/docs） | 极简黑白配色、清晰的信息层级、代码块样式、导航设计 |
| **Tailwind CSS 文档**（tailwindcss.com） | 侧边栏导航结构、搜索交互（Cmd+K）、代码片段展示方式 |
| **Linear**（linear.app） | 暗色模式设计语言、微动效（hover 效果、过渡动画）、整体的"高级感" |
| **Raycast**（raycast.com） | 开发者工具的品牌调性、卡片设计、图标风格 |
| **Stripe 文档**（stripe.com/docs） | 内容的结构化呈现、代码/文档混排方式、Tab 切换交互 |

### 品牌调性关键词

**主调性**：
- **专业克制**（Professional & Restrained）：不炫技，用设计服务内容
- **高效聚焦**（Efficient & Focused）：用户来找答案，设计不制造干扰
- **现代技术感**（Modern & Technical）：等宽字体点缀、代码风格元素、暗色主题

**辅助调性**：
- **可信赖**（Trustworthy）：来自实战经验，非纸上谈兵
- **结构化**（Structured）：信息有条理，导航有逻辑
- **开放分享**（Open & Sharing）：社区驱动，知识共享

### 配色方向

- **主色调**：深色系（接近 #0A0A0A ~ #1A1A2E 的深色背景），搭配高亮强调色
- **强调色**：冷色调蓝/紫/青中选择一个，用于链接、按钮、高亮标记（如 #3B82F6 蓝 或 #8B5CF6 紫 或 #06B6D4 青）
- **文字色**：主文字高对比白（#F5F5F5），次级文字灰（#A3A3A3）
- **代码块**：深色背景 + 语法高亮配色（类 VS Code Dark+ 或 One Dark 方案）
- **支持亮色模式**：白色背景（#FAFAFA）+ 深色文字（#171717）的反转方案

### 字体方向

- **标题**：Inter / Plus Jakarta Sans 等现代无衬线体
- **正文**：Inter / system-ui，强调可读性
- **代码**：JetBrains Mono / Fira Code 等宽字体
- **中文**：系统默认字体栈（-apple-system, "PingFang SC", "Microsoft YaHei"）

### 间距与布局

- 大量留白，内容区最大宽度 1200px 居中
- 卡片使用微弱边框 + 半透明背景，而非强投影
- 模块间间距充裕（64px-96px），段落间距适中（24px-32px）

### 动效基调

- **克制微动效**：hover 状态变化（背景色/边框色过渡，150ms-200ms）
- **入场动画**：页面内容从下方微微浮入（translateY: 10px → 0，200ms-300ms ease-out）
- **避免**：大面积动画、弹跳效果、长时间加载动画
- **搜索面板**：Cmd+K 弹出时有 backdrop-blur 毛玻璃效果

## 7. 范围边界

### MVP 包含
- 首页（Hero + 模块入口卡片）
- Skills 列表页（卡片展示 + 场景分类筛选）
- Skill 详情页（完整信息 + 代码一键复制）
- 开发流程总览页（流程总览图 + 7 场景入口）
- 场景详情页（步骤流程 + Skills 推荐）
- 经验总结列表页和详情页（Markdown 渲染）
- 全局导航栏 + 全局搜索（Cmd+K）
- 暗色/亮色模式切换
- 响应式布局（桌面 + 移动端）
- 静态站点部署（Vercel / Netlify / GitHub Pages）

### MVP 不包含（后续迭代）
- 用户账号体系与个人收藏功能
- Skills 社区投稿与评分系统
- 评论与讨论功能
- 在线 Skill Playground（在线试运行 Skill）
- 多语言支持（英文版本）
- 流程决策树交互式引导
- 角色 Agent 展示页面（归入 v1.1）
- RSS 订阅和内容更新通知
- 数据统计和访问分析面板

### 明确排除
- 不做在线 IDE 或代码执行环境
- 不做 AI 对话/问答功能（不内嵌 LLM 对话）
- 不做用户生成内容（UGC）平台，内容由项目维护者管理
- 不做付费/商业化功能
- 不做原生移动 App
